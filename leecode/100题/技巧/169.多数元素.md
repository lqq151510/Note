**题目：多数元素**​

**描述**​

给定一个大小为 `n`的数组 `nums`，返回其中的多数元素。多数元素是指在数组中出现次数 **大于**​ `⌊ n/2 ⌋`的元素。

你可以假设数组是非空的，并且给定的数组总是存在多数元素。

**解题思路**​

由于多数元素出现的次数超过数组长度的一半，因此将数组排序后，中间位置的元素一定是多数元素。

- 时间复杂度：取决于排序算法，通常为 O(n log n)。
    
- 空间复杂度：取决于排序算法的实现，若使用原地排序则为 O(1)。
    

**代码实现（Java）**​

```
class Solution {
    public int majorityElement(int[] nums) {
        Arrays.sort(nums);
        int n = nums.length;
        return nums[n / 2];
    }
}
```

**复杂度分析**​

- 时间复杂度：O(n log n)，主要由排序决定。
    
- 空间复杂度：O(log n) 或 O(1)，取决于排序算法的空间开销（如 `Arrays.sort()`在 Java 中对于原始类型使用双轴快速排序，空间复杂度为 O(log n)）。
    

**注意**​

本题也可以使用 **摩尔投票法**​ 在 O(n) 时间和 O(1) 空间内解决，但排序方法更为直观且代码简洁。
