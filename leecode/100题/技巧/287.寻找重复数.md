**题目：寻找重复数**​

**描述**​

给定一个包含 `n + 1`个整数的数组 `nums`，其数字都在 `[1, n]`范围内（包括 1 和 n），可知至少存在一个重复的整数。

假设 `nums`只有 **一个重复的整数**，返回这个重复的数。

**要求**​

- 必须 **不修改**​ 数组 `nums`。
    
- 只能使用常量级 O(1) 的额外空间。
    

---

**解题思路（快慢指针 / Floyd 循环检测）**​

将数组视为一个链表，每个位置 `i`的值为 `nums[i]`，表示指向下一个节点的索引（即节点 `i`指向节点 `nums[i]`）。

- 由于数字范围是 `[1, n]`，且数组长度为 `n+1`，所以不会越界（但注意索引从 0 开始，而数字从 1 开始，不过因为 `nums[i]`作为索引时，实际指向的是 `nums[i]`这个值对应的位置，所以链表节点是 `0`到 `n`，共 `n+1`个节点）。
    
- 因为有重复数字，则至少有两个不同的索引指向同一个位置，从而形成环。重复的数字即为环的入口。
    

**算法步骤**​

1. **第一阶段**：检测环是否存在，并找到快慢指针的相遇点。
    
    - 初始化慢指针 `slow = 0`，快指针 `fast = 0`。
        
    - 慢指针每次走一步：`slow = nums[slow]`。
        
    - 快指针每次走两步：`fast = nums[nums[fast]]`。
        
    - 当 `slow == fast`时停止。
        
    
2. **第二阶段**：找到环的入口（即重复数字）。
    
    - 将慢指针重新置为 `0`。
        
    - 慢指针和快指针每次各走一步，直到再次相遇，相遇点即为环的入口（重复数字）。
        
    

**正确性证明**​

设从起点到环入口的距离为 `a`，环入口到相遇点的距离为 `b`，相遇点再到环入口的距离为 `c`，则有 `b + c = 环长`。

第一次相遇时，慢指针走了 `a + b`，快指针走了 `a + b + k*(b+c)`，且快指针速度是慢指针的两倍，所以 `2(a+b) = a + b + k*(b+c)`，解得 `a = (k-1)(b+c) + c`。

因此，从起点和相遇点同时同速出发，一定会在环入口相遇。

---

**代码实现（Java）**​

```
class Solution {
    public int findDuplicate(int[] nums) {
        int slow = 0;
        int fast = 0;
        // 第一阶段：找到相遇点
        do {
            slow = nums[slow];
            fast = nums[nums[fast]];
        } while (slow != fast);
        
        // 第二阶段：找到环入口
        slow = 0;
        while (slow != fast) {
            slow = nums[slow];
            fast = nums[fast];
        }
        return slow; // 或 fast
    }
}
```

**复杂度分析**​

- 时间复杂度：O(n)，其中 n 是数组长度。快慢指针最多遍历数组两次。
    
- 空间复杂度：O(1)，只使用了两个指针变量。
    

**注意**​

- 该算法不修改原数组，且满足常数空间要求。
    
- 由于题目保证只有一个重复数字，且数字范围在 [1, n] 内，此方法有效。若存在多个重复，则需另作处理。
