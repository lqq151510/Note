### 题目描述

给定一个非负整数数组 `nums`，你最初位于数组的第一个下标。数组中的每个元素代表你在该位置可以跳跃的最大长度。判断你是否能够到达最后一个下标，如果可以，返回 `true`；否则，返回 `false`。

---

### 解题思路

使用贪心算法，遍历数组并记录当前能够到达的最远位置（`maxReach`）：

1. 如果当前位置索引大于之前能够到达的最远位置，则无法继续前进，返回 `false`。
    
2. 更新当前能够到达的最远位置为 `max(maxReach, i + nums[i])`。
    
3. 如果最远位置已经能覆盖最后一个下标，提前返回 `true`。
    

---

### 代码实现

```
class Solution {
    public boolean canJump(int[] nums) {
        if (nums == null || nums.length <= 1) {
            return true;
        }
        int maxReach = 0; // 当前能够到达的最远位置
        int n = nums.length;
        for (int i = 0; i < n; i++) {
            // 如果当前位置已经超出之前能够到达的最远位置，则无法继续
            if (i > maxReach) {
                return false;
            }
            // 更新能够到达的最远位置
            maxReach = Math.max(maxReach, i + nums[i]);
            // 如果已经可以到达最后一个位置，提前返回
            if (maxReach >= n - 1) {
                return true;
            }
        }
        return true;
    }
}
```

---

### 复杂度分析

- **时间复杂度**：O(n)，只需遍历一次数组。
    
- **空间复杂度**：O(1)，只使用了常数个变量。
    

---

### 示例

```
输入: nums = [2,3,1,1,4]
输出: true
解释: 从位置 0 跳到 1（跳1步），然后从位置1跳3步到达最后一个位置。

输入: nums = [3,2,1,0,4]
输出: false
解释: 无论怎样，你总会到达索引为3的位置，但该位置的最大跳跃长度是0，所以永远不可能到达最后一个位置。
```
