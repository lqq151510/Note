## 问题描述

你是一个专业的小偷，计划偷窃沿街的房屋。每间房屋内都藏有一定金额的现金，但相邻的房屋装有相互连通的防盗系统。如果两间相邻的房屋在同一晚被闯入，系统会自动报警。

给定一个代表每个房屋存放金额的非负整数数组，请计算在不触动警报装置的情况下，一夜之内能够偷窃到的最高金额。

## 解题思路

这是一个典型的动态规划问题，可以使用以下递推关系解决：

- 定义 `dp[i]`表示偷窃到第 `i`个房屋时能获得的最大金额。
    
- 对于第 `i`个房屋，有两种选择：
    
    1. 偷窃第 `i`个房屋，则不能偷窃第 `i-1`个房屋，最大金额为 `dp[i-2] + nums[i]`。
        
    2. 不偷窃第 `i`个房屋，则最大金额为 `dp[i-1]`。
        
    
- 取两者中的较大值作为 `dp[i]`的值。
    

**状态转移方程**：

```
dp[i] = max(dp[i-2] + nums[i], dp[i-1])
```

**初始化**：

- 如果只有一个房屋，只能偷窃该房屋：`dp[0] = nums[0]`。
    
- 如果有两个房屋，由于不能同时偷窃相邻房屋，选择金额较大的一个：`dp[1] = max(nums[0], nums[1])`。
    

**结果**：

- 最终结果为 `dp[n-1]`，其中 `n`是房屋数量。
    

## 复杂度分析

- **时间复杂度**：O(n)，其中 n 是数组长度，只需遍历一次数组。
    
- **空间复杂度**：O(n)，用于存储动态规划数组。可以优化为 O(1)，只保留前两个状态。
    

## 代码实现（带注释）

```
class Solution {
    public int rob(int[] nums) {
        // 处理边界情况
        if (nums == null || nums.length == 0) {
            return 0;
        }
        if (nums.length == 1) {
            return nums[0];
        }
        
        // dp[i] 表示偷窃到第 i 个房屋时的最大金额
        int[] dp = new int[nums.length];
        
        // 初始化
        dp[0] = nums[0]; // 只有一间房屋时，只能偷这一间
        dp[1] = Math.max(nums[0], nums[1]); // 两间房屋时，选择金额较大的偷窃
        
        // 状态转移
        for (int i = 2; i < nums.length; i++) {
            // 选择偷窃当前房屋：则不能偷窃前一间房屋，金额为 dp[i-2] + nums[i]
            // 选择不偷窃当前房屋：则金额为 dp[i-1]
            dp[i] = Math.max(dp[i - 2] + nums[i], dp[i - 1]);
        }
        
        return dp[nums.length - 1];
    }
}
```

## 空间优化版本

实际上，我们只需要保存前两个状态即可，可以将空间复杂度优化为 O(1)：

```
class Solution {
    public int rob(int[] nums) {
        if (nums == null || nums.length == 0) {
            return 0;
        }
        if (nums.length == 1) {
            return nums[0];
        }
        
        int prev2 = nums[0]; // 代表 dp[i-2]
        int prev1 = Math.max(nums[0], nums[1]); // 代表 dp[i-1]
        
        for (int i = 2; i < nums.length; i++) {
            int current = Math.max(prev2 + nums[i], prev1);
            prev2 = prev1;
            prev1 = current;
        }
        
        return prev1;
    }
}
```

这样，代码更加高效，且易于理解。
