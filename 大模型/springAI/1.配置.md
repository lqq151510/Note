Spring AI 的配置主要分为**依赖配置**和**应用配置**两部分。由于 Spring AI 1.0 版本已发布，配置方式与早期版本有较大差异，以下是基于当前稳定版的详细配置指南。

### 一、环境与依赖配置

#### 1. 环境要求

- **JDK 版本**：推荐 **JDK 17+**（Spring AI 1.0 强制要求）。
    
- **Spring Boot 版本**：推荐 **Spring Boot 3.2.x**​ 或更高版本。
    

#### 2. 依赖管理（BOM）

Spring AI 1.0 引入了 BOM（Bill of Materials）来统一管理所有模块的版本，这是配置的第一步，能有效避免版本冲突。

**Maven 配置（pom.xml）**

```
<dependencyManagement>
    <dependencies>
        <!-- 引入 Spring AI BOM 统一版本 -->
        <dependency>
            <groupId>org.springframework.ai</groupId>
            <artifactId>spring-ai-bom</artifactId>
            <version>1.0.0</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

**Gradle 配置（build.gradle）**

```
dependencies {
    // 引入 BOM
    implementation platform('org.springframework.ai:spring-ai-bom:1.0.0')
}
```

#### 3. 核心依赖（Starter）

根据你使用的模型提供商选择对应的 Starter。以 **OpenAI**​ 和 **DeepSeek**（兼容 OpenAI API）为例：

**Maven 配置**

```
<dependencies>
    <!-- Spring Boot Web 基础依赖 -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    
    <!-- Spring AI OpenAI Starter (支持 OpenAI 及兼容 API 的模型，如 DeepSeek) -->
    <dependency>
        <groupId>org.springframework.ai</groupId>
        <artifactId>spring-ai-openai-spring-boot-starter</artifactId>
    </dependency>
</dependencies>
```

**Gradle 配置**

```
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    implementation 'org.springframework.ai:spring-ai-openai-spring-boot-starter'
}
```

### 二、应用配置（application.yml / application.properties）

配置文件的格式取决于你使用的模型。**OpenAI 兼容模型**（如 DeepSeek、智谱、通义千问等）通常使用 `spring.ai.openai`前缀。

#### 1. 基础配置（YAML 格式）

```
spring:
  ai:
    openai:
      # 1. API 密钥（建议使用环境变量）
      api-key: ${OPENAI_API_KEY:sk-xxx} # 冒号后为默认值，生产环境建议使用环境变量
      # 2. 模型名称
      chat:
        options:
          model: gpt-3.5-turbo # 或 deepseek-chat, qwen-turbo 等
          temperature: 0.7
          max-tokens: 2000
```

#### 2. 国内模型配置（以 DeepSeek 为例）

国内模型通常需要修改 `base-url`指向其专属 API 端点。

```
spring:
  ai:
    openai:
      # 修改为国内模型的 API 地址
      base-url: https://api.deepseek.com/v1
      api-key: sk-xxx # 替换为 DeepSeek 的 API Key
      chat:
        options:
          model: deepseek-chat # DeepSeek 模型名称
          temperature: 0.7
```

#### 3. 多模型配置（同一供应商）

如果你需要配置同一供应商下的多个模型（例如同时使用 GPT-4 和 GPT-3.5），Spring AI 1.0 支持通过 `@Qualifier`注入不同的 `ChatClient`。

**配置文件示例**：

```
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY}
      # 主模型配置
      chat:
        options:
          model: gpt-4
      # 辅助模型配置（自定义属性）
      secondary:
        model: gpt-3.5-turbo
```

**Java 配置类**：

```
@Configuration
public class MultiModelConfig {
    
    @Bean
    @Primary
    public ChatClient primaryChatClient(OpenAiChatModel chatModel) {
        return ChatClient.create(chatModel);
    }
    
    @Bean
    public ChatClient secondaryChatClient(
            OpenAiApi openAiApi,
            @Value("${spring.ai.openai.secondary.model}") String modelName) {
        // 手动构建第二个 ChatModel
        OpenAiChatModel secondaryModel = new OpenAiChatModel(openAiApi,
            OpenAiChatOptions.builder()
                .withModel(modelName)
                .build());
        return ChatClient.create(secondaryModel);
    }
}
```

### 三、代码使用示例

配置完成后，你可以通过注入 `ChatClient`来调用 AI 服务。

```
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.stereotype.Service;

@Service
public class AIService {
    
    private final ChatClient chatClient;
    
    public AIService(ChatClient chatClient) {
        this.chatClient = chatClient;
    }
    
    public String chat(String message) {
        return chatClient.prompt()
            .user(message)
            .call()
            .content();
    }
}
```

### 四、常见问题与注意事项

1. **版本兼容性**：确保 `spring-ai-bom`的版本与你的 Spring Boot 版本兼容。Spring AI 1.0 要求 Spring Boot 3.2+。
    
2. **API Key 安全**：**切勿将 API Key 硬编码在配置文件中**。建议使用环境变量（`${API_KEY}`）或 Spring Cloud Config 等配置中心管理。
    
3. **网络代理**：如果部署在内网环境需要访问外网 API，请确保网络代理配置正确，或使用国内模型（如 DeepSeek）避免网络问题。
    

通过以上配置，你的 Spring Boot 应用即可快速集成 AI 能力。
