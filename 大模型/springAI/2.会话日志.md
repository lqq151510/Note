Spring AI 自身不提供内置的、开箱即用的持久化会话日志功能。它主要专注于 AI 模型的交互抽象和便捷调用。

要实现会话日志的**记录、存储和查询**，需要**结合其他技术**来设计和实现。以下是几种主流的实现方案：

### 方案一：手动记录到数据库（推荐）

这是最灵活和可控的方案，你可以完全定义日志的结构、存储方式和查询逻辑。

**1. 创建日志实体（JPA 示例）**

```
import jakarta.persistence.*;
import java.time.LocalDateTime;

@Entity
@Table(name = "chat_log")
public class ChatLog {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String sessionId; // 用于关联同一会话
    private String role;      // "user" 或 "assistant"
    
    @Column(columnDefinition = "TEXT")
    private String content;   // 消息内容
    
    private String model;     // 使用的模型，如 gpt-3.5-turbo
    private Integer promptTokens;
    private Integer completionTokens;
    private Integer totalTokens;
    private LocalDateTime timestamp;
    
    // 省略构造函数、Getter/Setter
}
```

**2. 创建 Repository**

```
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Query;
import java.util.List;

public interface ChatLogRepository extends JpaRepository<ChatLog, Long> {
    List<ChatLog> findBySessionIdOrderByTimestampAsc(String sessionId);
}
```

**3. 在调用 AI 服务前后进行记录**

你可以创建一个 **AOP 切面**​ 或一个 **包装服务**​ 来统一处理日志记录。

```
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.model.ChatResponse;
import org.springframework.ai.chat.prompt.Prompt;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.time.LocalDateTime;
import java.util.UUID;

@Service
public class LoggingAIService {
    
    private final ChatClient chatClient;
    private final ChatLogRepository logRepository;
    
    public LoggingAIService(ChatClient chatClient, ChatLogRepository logRepository) {
        this.chatClient = chatClient;
        this.logRepository = logRepository;
    }
    
    @Transactional
    public String chatWithLogging(String sessionId, String userMessage) {
        // 1. 记录用户输入
        ChatLog userLog = new ChatLog();
        userLog.setSessionId(sessionId);
        userLog.setRole("user");
        userLog.setContent(userMessage);
        userLog.setTimestamp(LocalDateTime.now());
        logRepository.save(userLog);
        
        // 2. 调用 AI
        ChatResponse response = chatClient.prompt()
            .user(userMessage)
            .call()
            .chatResponse();
        
        String aiReply = response.getResult().getOutput().getContent();
        
        // 3. 记录 AI 回复
        ChatLog aiLog = new ChatLog();
        aiLog.setSessionId(sessionId);
        aiLog.setRole("assistant");
        aiLog.setContent(aiReply);
        aiLog.setTimestamp(LocalDateTime.now());
        // 可记录 Token 使用量（如果模型返回）
        if (response.getUsage() != null) {
            aiLog.setPromptTokens(response.getUsage().getPromptTokens());
            aiLog.setCompletionTokens(response.getUsage().getCompletionTokens());
            aiLog.setTotalTokens(response.getUsage().getTotalTokens());
        }
        logRepository.save(aiLog);
        
        return aiReply;
    }
    
    // 生成或获取会话 ID
    public String generateSessionId() {
        return UUID.randomUUID().toString();
    }
    
    // 查询某个会话的历史
    public List<ChatLog> getSessionHistory(String sessionId) {
        return logRepository.findBySessionIdOrderByTimestampAsc(sessionId);
    }
}
```

**4. 在 Controller 中使用**

```
import org.springframework.web.bind.annotation.*;
import java.util.List;

@RestController
@RequestMapping("/api/ai")
public class AIController {
    private final LoggingAIService aiService;
    
    public AIController(LoggingAIService aiService) {
        this.aiService = aiService;
    }
    
    @PostMapping("/chat")
    public ChatResult chat(@RequestBody ChatRequest request) {
        String reply = aiService.chatWithLogging(request.getSessionId(), request.getMessage());
        return new ChatResult(reply);
    }
    
    @GetMapping("/history/{sessionId}")
    public List<ChatLog> getHistory(@PathVariable String sessionId) {
        return aiService.getSessionHistory(sessionId);
    }
}
// 省略 ChatRequest, ChatResult 等 DTO
```

### 方案二：集成向量数据库（用于语义检索）

如果你的日志需要支持**基于语义的搜索**（例如，查找与某个问题相似的历史对话），可以考虑使用向量数据库。

- **常用选择**：`Spring AI`对 **PgVector**（PostgreSQL 扩展）、**Redis**、**Milvus**​ 等有良好的支持。
    
- **实现方式**：
    
    1. 在记录日志时，使用 `EmbeddingModel`将消息内容转换为向量。
        
    2. 将**向量**和**文本**一起存储到向量数据库中。
        
    3. 查询时，先将查询问题向量化，然后执行相似度搜索。
        
    

这方案更复杂，适用于需要智能检索历史对话的场景。

### 方案三：使用 Observability 框架（用于监控）

如果你主要关心**性能监控、调用链追踪和指标收集**，而不是内容本身，可以集成以下框架：

- **Spring Boot Actuator**：暴露 AI 调用的健康状态和指标（如调用次数、耗时）。
    
- **Micrometer Tracing**：与分布式追踪系统（如 Zipkin, Jaeger）集成，可视化 AI 调用的延迟和依赖关系。
    

这适用于运维和性能分析场景。

### 核心总结

|方案|适用场景|优点|缺点|
|---|---|---|---|
|**手动数据库记录**​|需要持久化、审计、简单回查对话历史|结构灵活，查询简单，数据完全可控|需要自行设计和实现|
|**向量数据库**​|需要基于语义搜索历史对话|支持智能检索，功能强大|架构复杂，维护成本高|
|**Observability**​|关注性能监控、调用链，而非对话内容|标准化，易于集成到现有监控体系|不记录具体的对话内容|

**推荐实践**：对于大多数业务应用，**方案一（手动记录到数据库）**​ 是最直接、最可靠的选择。你可以在其中扩展字段，记录用户ID、请求IP、成本等信息，满足审计、分析和上下文管理的多种需求。
