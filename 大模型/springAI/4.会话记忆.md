Spring AI 提供了**内置的会话记忆（Memory）功能**，但它是上下文级的记忆，而非持久化的长期记忆。以下是完整的会话记忆实现方案，包含**内置记忆、持久化记忆和上下文管理**。

---

## 一、Spring AI 内置记忆（上下文窗口）

Spring AI 提供了 `ChatMemory`接口及其实现，用于在**单次对话中**维护上下文。

### 1. 基础配置

```
# application.yml
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY}
      chat:
        options:
          model: gpt-3.5-turbo
          # 上下文记忆参数
          max-context-messages: 20  # 最大记忆消息数
```

### 2. 使用 ChatClient 自带记忆

```
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.client.advisor.MessageChatMemoryAdvisor;
import org.springframework.ai.chat.memory.ChatMemory;
import org.springframework.ai.chat.memory.InMemoryChatMemory;
import org.springframework.stereotype.Service;
import org.springframework.web.context.annotation.SessionScope;
import java.util.List;

@Service
@SessionScope  // 每个会话独立实例
public class SessionAIService {
    
    private final ChatClient chatClient;
    private final ChatMemory chatMemory;
    private final String sessionId;
    
    public SessionAIService(ChatClient.Builder chatClientBuilder) {
        this.sessionId = UUID.randomUUID().toString();
        
        // 创建基于内存的聊天记忆，每个会话独立
        this.chatMemory = new InMemoryChatMemory();
        
        // 构建带记忆的 ChatClient
        this.chatClient = chatClientBuilder
            .defaultAdvisors(
                new MessageChatMemoryAdvisor(this.chatMemory)  // 添加记忆顾问
            )
            .defaultSystem("你是一个有用的助手")
            .build();
    }
    
    public String chat(String userMessage) {
        return chatClient.prompt()
            .user(userMessage)
            .call()
            .content();
    }
    
    // 获取当前会话的所有记忆
    public List<org.springframework.ai.chat.messages.Message> getConversationHistory() {
        return chatMemory.getConversationHistory();
    }
    
    // 清除当前会话记忆
    public void clearMemory() {
        chatMemory.clear();
    }
    
    public String getSessionId() {
        return sessionId;
    }
}
```

## 二、持久化会话记忆实现（推荐）

基于数据库实现长期记忆，支持会话重启后恢复历史。

### 1. 数据库表设计

```
-- 会话表
CREATE TABLE ai_session (
    id VARCHAR(64) PRIMARY KEY,
    user_id VARCHAR(64),           -- 关联用户
    title VARCHAR(255),           -- 会话标题（第一条消息生成）
    model VARCHAR(50),            -- 使用的模型
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user (user_id),
    INDEX idx_created (created_at)
);

-- 消息表（存储完整的对话上下文）
CREATE TABLE ai_conversation (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    session_id VARCHAR(64) NOT NULL,
    message_index INT NOT NULL,    -- 消息在会话中的顺序
    role VARCHAR(20) NOT NULL,    -- 'system', 'user', 'assistant'
    content TEXT NOT NULL,         -- 消息内容
    tokens INT,                    -- 消息消耗的token数
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_session_message (session_id, message_index),
    FOREIGN KEY (session_id) REFERENCES ai_session(id) ON DELETE CASCADE,
    INDEX idx_session (session_id)
);

-- 消息摘要表（用于长对话优化）
CREATE TABLE ai_conversation_summary (
    session_id VARCHAR(64) PRIMARY KEY,
    summary TEXT,                  -- 对话摘要
    message_count INT,             -- 已总结的消息数
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (session_id) REFERENCES ai_session(id) ON DELETE CASCADE
);
```

### 3. 智能记忆管理服务

```
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.model.ChatResponse;
import org.springframework.ai.chat.model.Usage;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.util.*;
import java.util.stream.Collectors;

@Service
public class SmartMemoryService {
    
    private final ChatClient chatClient;
    private final AISessionRepository sessionRepository;
    private final ConversationMessageRepository messageRepository;
    private final ConversationSummaryRepository summaryRepository;
    
    // 配置参数
    private static final int MAX_CONTEXT_MESSAGES = 20;  // 最大上下文消息数
    private static final int SUMMARY_THRESHOLD = 10;     // 触发总结的对话轮数
    
    public SmartMemoryService(ChatClient chatClient,
                             AISessionRepository sessionRepository,
                             ConversationMessageRepository messageRepository,
                             ConversationSummaryRepository summaryRepository) {
        this.chatClient = chatClient;
        this.sessionRepository = sessionRepository;
        this.messageRepository = messageRepository;
        this.summaryRepository = summaryRepository;
    }
    
    /**
     * 智能对话（带记忆管理）
     */
    @Transactional
    public ChatResult chatWithMemory(String sessionId, String userId, String userMessage) {
        // 1. 获取或创建会话
        AISession session = getOrCreateSession(sessionId, userId);
        
        // 2. 如果是新会话的第一条消息，生成会话标题
        if (session.getMessages().isEmpty()) {
            String title = generateConversationTitle(userMessage);
            session.setTitle(title);
            sessionRepository.save(session);
        }
        
        // 3. 保存用户消息
        ConversationMessage userMsg = saveMessage(session, MessageRole.USER, userMessage, null);
        
        // 4. 构建带上下文的prompt
        String contextPrompt = buildContextPrompt(session);
        
        // 5. 调用AI
        ChatResponse response = chatClient.prompt()
            .system(contextPrompt)  // 包含历史摘要和最近对话
            .user(userMessage)
            .call()
            .chatResponse();
        
        String aiReply = response.getResult().getOutput().getContent();
        
        // 6. 保存AI回复
        ConversationMessage aiMsg = saveMessage(session, MessageRole.ASSISTANT, aiReply, response.getUsage());
        
        // 7. 检查是否需要生成摘要
        if (shouldGenerateSummary(session)) {
            generateAndSaveSummary(session);
        }
        
        // 8. 返回结果
        ChatResult result = new ChatResult();
        result.setReply(aiReply);
        result.setSessionId(session.getId());
        result.setTitle(session.getTitle());
        result.setUsage(toUsageDto(response.getUsage()));
        
        return result;
    }
    
    /**
     * 构建智能上下文Prompt
     * 策略：摘要 + 最近对话
     */
    private String buildContextPrompt(AISession session) {
        StringBuilder context = new StringBuilder();
        
        // 1. 添加对话摘要
        ConversationSummary summary = summaryRepository.findById(session.getId()).orElse(null);
        if (summary != null && summary.getSummary() != null) {
            context.append("以下是之前对话的摘要：\n")
                   .append(summary.getSummary())
                   .append("\n\n");
        }
        
        // 2. 添加最近的对话
        List<ConversationMessage> recentMessages = getRecentMessages(session, MAX_CONTEXT_MESSAGES);
        for (ConversationMessage msg : recentMessages) {
            String role = msg.getRole() == MessageRole.USER ? "用户" : "助手";
            context.append(role).append(": ").append(msg.getContent()).append("\n");
        }
        
        return context.toString();
    }
    
    /**
     * 生成对话摘要
     */
    private void generateAndSaveSummary(AISession session) {
        // 获取需要总结的旧消息
        List<ConversationMessage> oldMessages = getOldMessagesForSummary(session);
        
        if (oldMessages.isEmpty()) {
            return;
        }
        
        // 构建总结提示
        String conversationText = oldMessages.stream()
            .map(msg -> msg.getRole().name() + ": " + msg.getContent())
            .collect(Collectors.joining("\n"));
        
        String summaryPrompt = String.format("""
            请将以下对话总结为简洁的摘要，保留重要信息和细节：
            
            %s
            
            摘要：
            """, conversationText);
        
        // 调用AI生成摘要
        String summary = chatClient.prompt()
            .system("你是一个专业的对话总结助手")
            .user(summaryPrompt)
            .call()
            .content();
        
        // 保存摘要
        ConversationSummary convSummary = summaryRepository.findById(session.getId())
            .orElse(new ConversationSummary());
        convSummary.setSession(session);
        convSummary.setSummary(summary);
        convSummary.setMessageCount(oldMessages.size());
        summaryRepository.save(convSummary);
        
        // 可选：删除已总结的旧消息以减少存储
        // messageRepository.deleteAll(oldMessages);
    }
    
    /**
     * 获取最近的消息
     */
    private List<ConversationMessage> getRecentMessages(AISession session, int limit) {
        List<ConversationMessage> allMessages = session.getMessages();
        int start = Math.max(0, allMessages.size() - limit);
        return allMessages.subList(start, allMessages.size());
    }
    
    /**
     * 获取需要总结的旧消息
     */
    private List<ConversationMessage> getOldMessagesForSummary(AISession session) {
        List<ConversationMessage> allMessages = session.getMessages();
        ConversationSummary summary = summaryRepository.findById(session.getId()).orElse(null);
        
        int startIndex = 0;
        if (summary != null) {
            startIndex = summary.getMessageCount();
        }
        
        int endIndex = Math.max(0, allMessages.size() - 5); // 保留最后5条消息
        if (startIndex >= endIndex) {
            return Collections.emptyList();
        }
        
        return allMessages.subList(startIndex, endIndex);
    }
    
    private boolean shouldGenerateSummary(AISession session) {
        ConversationSummary summary = summaryRepository.findById(session.getId()).orElse(null);
        int unsummarizedCount = session.getMessages().size() - 
                               (summary != null ? summary.getMessageCount() : 0);
        return unsummarizedCount >= SUMMARY_THRESHOLD;
    }
    
    // 其他辅助方法...
    private AISession getOrCreateSession(String sessionId, String userId) { /* ... */ }
    private ConversationMessage saveMessage(AISession session, MessageRole role, String content, Usage usage) { /* ... */ }
    private String generateConversationTitle(String firstMessage) { /* ... */ }
}

## 三、高级记忆策略

### 1. Token 感知的记忆窗口

```
@Component
public class TokenAwareMemory {
    
    private final ChatClient chatClient;
    private final TokenCountEstimator tokenEstimator;
    private final int MAX_TOKENS = 4000;  // 模型上下文限制
    
    public List<Message> getOptimalContext(List<ConversationMessage> allMessages, 
                                          ConversationSummary summary) {
        List<Message> context = new ArrayList<>();
        int totalTokens = 0;
        
        // 1. 添加系统提示
        String systemPrompt = buildSystemPrompt(summary);
        int systemTokens = tokenEstimator.estimate(systemPrompt);
        totalTokens += systemTokens;
        
        // 2. 从新到旧添加消息，直到达到token限制
        for (int i = allMessages.size() - 1; i >= 0; i--) {
            ConversationMessage msg = allMessages.get(i);
            int msgTokens = msg.getTokens() != null ? msg.getTokens() : 
                           tokenEstimator.estimate(msg.getContent());
            
            if (totalTokens + msgTokens > MAX_TOKENS) {
                break;
            }
            
            context.add(0, convertToAIMessage(msg));  // 保持时间顺序
            totalTokens += msgTokens;
        }
        
        return context;
    }
    
    // 使用tiktoken或类似库估算token
    @Component
    public static class TokenCountEstimator {
        public int estimate(String text) {
            // 简单估算：1个中文字符 ≈ 1.3个token，1个英文单词 ≈ 1.3个token
            int chineseChars = (int) text.chars().filter(c -> c >= 0x4E00 && c <= 0x9FFF).count();
            int otherChars = text.length() - chineseChars;
            return (int) Math.ceil(chineseChars * 1.3 + otherChars * 0.8);
        }
    }
}

### 2. 向量记忆检索（长期记忆）

```
@Component
public class VectorMemoryService {
    
    private final EmbeddingModel embeddingModel;
    private final VectorStore vectorStore;
    
    // 存储重要信息到向量数据库
    public void storeMemory(String sessionId, String text, Map<String, Object> metadata) {
        // 1. 将文本转换为向量
        List<Double> embedding = embeddingModel.embed(text);
        
        // 2. 创建文档
        Document document = new Document(text, metadata);
        document.setEmbedding(embedding);
        
        // 3. 存储到向量数据库
        vectorStore.add(List.of(document));
    }
    
    // 检索相关记忆
    public List<String> retrieveRelevantMemories(String sessionId, String query, int topK) {
        // 1. 将查询转换为向量
        List<Double> queryEmbedding = embeddingModel.embed(query);
        
        // 2. 语义搜索
        List<Document> results = vectorStore.similaritySearch(queryEmbedding, topK);
        
        // 3. 过滤和格式化
        return results.stream()
            .filter(doc -> doc.getMetadata().get("sessionId").equals(sessionId))
            .map(Document::getContent)
            .collect(Collectors.toList());
    }
    
    // 在对话中使用向量记忆
    public String chatWithLongTermMemory(String sessionId, String userMessage) {
        // 1. 检索相关长期记忆
        List<String> relevantMemories = retrieveRelevantMemories(sessionId, userMessage, 3);
        
        // 2. 构建增强的prompt
        String memoryContext = "";
        if (!relevantMemories.isEmpty()) {
            memoryContext = "相关背景信息：\n" + 
                String.join("\n", relevantMemories) + "\n\n";
        }
        
        // 3. 获取最近的对话上下文
        String recentContext = getRecentConversation(sessionId);
        
        // 4. 组合调用
        String prompt = memoryContext + recentContext + "\n用户：" + userMessage;
        
        return chatClient.prompt()
            .user(prompt)
            .call()
            .content();
    }
}

## 六、前端集成示例

```
// 带记忆的聊天组件
class MemoryChat {
    constructor(userId, apiBase = '/api/ai/memory') {
        this.userId = userId;
        this.sessionId = localStorage.getItem('ai_session_id') || null;
        this.apiBase = apiBase;
    }
    
    // 发送消息
    async sendMessage(message) {
        const response = await fetch(`${this.apiBase}/chat`, {
            method: 'POST',
            headers: {'Content-Type': 'application/json'},
            body: JSON.stringify({
                sessionId: this.sessionId,
                userId: this.userId,
                message: message
            })
        });
        
        const data = await response.json();
        
        // 保存会话ID
        if (data.sessionId && !this.sessionId) {
            this.sessionId = data.sessionId;
            localStorage.setItem('ai_session_id', this.sessionId);
        }
        
        return data;
    }
    
    // 获取会话历史
    async getHistory() {
        if (!this.sessionId) return [];
        
        const response = await fetch(`${this.apiBase}/session/${this.sessionId}`);
        return await response.json();
    }
    
    // 获取所有会话
    async getSessions() {
        const response = await fetch(`${this.apiBase}/sessions?userId=${this.userId}`);
        return await response.json();
    }
    
    // 切换会话
    switchSession(sessionId) {
        this.sessionId = sessionId;
        localStorage.setItem('ai_session_id', sessionId);
    }
}

## 关键点总结

1. **分层记忆策略**：
    
    - 短期记忆：最近N条对话（内存）
        
    - 中期记忆：对话摘要（数据库）
        
    - 长期记忆：关键信息向量检索（向量数据库）
        
    
2. **Token优化**：
    
    - 根据token数量动态调整上下文窗口
        
    - 定期生成摘要减少token消耗
        
    - 使用向量检索代替完整历史
        
    
3. **性能考虑**：
    
    - 热点会话缓存到Redis
        
    - 异步生成摘要
        
    - 分页加载历史记录
        
    
4. **扩展性**：
    
    - 支持多用户隔离
        
    - 可配置的记忆策略
        
    - 支持多种向量存储后端
        
    

这种实现方式可以支持上千轮的对话，同时保持高性能和合理的token消耗。
