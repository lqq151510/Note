Redis 最核心、最常用的 5 种数据类型分别是：**String（字符串）**、**Hash（哈希）**、**List（列表）**、**Set（集合）** 和 **ZSet（有序集合）**。

以下是这 5 种数据类型的详细解析和常见应用场景：

### 1. String（字符串）

这是 Redis 最基本的数据类型。它是一个键值对（Key-Value），其中 Value 不仅可以是字符串，也可以是数字（整数或浮点数），甚至是序列化后的图片或 JSON 对象。

- **特点：** 二进制安全，最大可以存储 512MB 的数据。支持对数字进行原子增减操作。
    
- **常见应用场景：**
    
    - **常规缓存：** 缓存高频访问的数据库数据（如商品信息、配置数据）或 Session。
        
    - **计数器：** 统计文章阅读量、视频播放量、微博点赞数（利用 `INCR` 和 `DECR` 命令）。
        
    - **分布式锁：** 利用 `SETNX` 命令实现简单的分布式锁。
        

### 2. Hash（哈希）

Hash 是一个键值对集合，它的 Value 本身又是一个类似 Map（field-value）的结构。它特别适合用于存储对象。

- **特点：** 相比于把整个对象序列化成 String 存储，Hash 可以直接对对象中的某个字段进行读取和修改，效率更高，也更节省内存。
    
- **常见应用场景：**
    
    - **对象存储：** 存储用户信息（如姓名、年龄、头像等）、商品详情等。
        
    - **购物车：** Key 是用户 ID，Field 是商品 ID，Value 是商品数量。
        

### 3. List（列表）

List 是简单的字符串列表，按照插入顺序排序。它的底层实现是一个双向链表。

- **特点：** 可以在列表的头部（左边）或尾部（右边）添加元素。支持通过索引获取元素，也支持阻塞弹出操作。
    
- **常见应用场景：**
    
    - **消息队列：** 利用 `LPUSH`（左侧推入）和 `RPOP`（右侧弹出）可以实现简单的异步消息队列。
        
    - **时间线 / 最新列表：** 比如朋友圈动态、最新评论列表、最新文章展示等，保证按时间顺序倒序排列。
        

### 4. Set（集合）

Set 是无序且**不重复**的字符串集合。

- **特点：** 提供了判断某个元素是否在集合内的强大功能，同时支持集合之间的聚合计算（如求交集、并集、差集）。
    
- **常见应用场景：**
    
    - **标签系统：** 给用户打标签（如“科技迷”、“运动爱好者”），或者给文章打标签。
        
    - **社交网络关系：** 利用交集求“共同好友”或“共同关注”的人。
        
    - **抽奖活动：** 利用 `SRANDMEMBER`（随机获取不删除）或 `SPOP`（随机弹出并删除）实现随机抽奖。
        

### 5. ZSet / Sorted Set（有序集合）

ZSet 和 Set 类似，也是不重复的字符串集合。不同的是，ZSet 中的每个元素都会关联一个 `double` 类型的浮点数，称为分数（score）。Redis 就是通过这个分数对集合中的元素进行从小到大的排序。

- **特点：** 元素是唯一的，但分数（score）可以重复。不仅可以按分数排序，还可以获取指定分数范围内的元素。
    
- **常见应用场景：**
    
    - **各类排行榜：** 如游戏积分排行榜、音乐播放量排行榜、热搜榜等。
        
    - **延时队列：** 将任务的执行时间戳作为 score 存入 ZSet，然后后台通过定时任务获取 score 小于当前时间戳的任务来执行。
        

---

**快速总结表：**

|**数据类型**|**核心特点**|**典型场景**|
|---|---|---|
|**String**|简单的 Key-Value，支持数字加减|缓存对象、计数器、分布式锁|
|**Hash**|适合存储对象的 Map 结构|用户信息、商品详情、购物车|
|**List**|有序的字符串列表（双向链表）|简单的消息队列、最新动态列表|
|**Set**|无序且不重复的集合，支持交/并/差集|标签系统、共同好友、抽奖|
|**ZSet**|带权重的有序集合，按分数排序|排行榜、延时队列|

除了这 5 种基础类型，Redis 后来还引入了 Bitmap（位图）、HyperLogLog（基数统计）、Geospatial（地理位置）等高级数据结构。
