Redis 的 **List（列表）** 是简单的字符串列表，按照插入顺序排序。你可以把元素添加到列表的头部（左边）或者尾部（右边）。它的底层实现类似于双向链表，因此在两端操作的速度非常快。

以下是常用的 Redis 列表操作命令汇总：

---

### 1. 添加元素 (Push)

这些命令用于将数据存入列表。

- **`LPUSH key value1 [value2]`**: 将一个或多个值插入到列表**头部**（左侧）。
    
- **`RPUSH key value1 [value2]`**: 将一个或多个值插入到列表**尾部**（右侧）。
    
- **`LPUSHX / RPUSHX key value`**: 只有当列表 **存在** 时，才插入数据（防止创建空列表）。
    

### 2. 弹出元素 (Pop)

从列表中移除并获取元素。

- **`LPOP key [count]`**: 移出并获取列表的第一个元素（左侧）。
    
- **`RPOP key [count]`**: 移出并获取列表的最后一个元素（右侧）。
    
- **`BLPOP / BRPOP key1 [key2] timeout`**: 移出并获取元素，如果列表没有元素会**阻塞**列表直到等待超时或发现可弹出元素为止（常用于消息队列）。
    

### 3. 查询与获取 (Query)

查看列表内容，但不一定会修改它。

- **`LRANGE key start stop`**: 获取列表指定范围内的元素。
    
    - _提示：`LRANGE mylist 0 -1` 可以查看列表中所有的元素。_
        
- **`LINDEX key index`**: 通过索引获取列表中的元素。
    
- **`LLEN key`**: 获取列表的长度。
    

### 4. 修改与删除 (Modify & Rem)

对现有数据进行清理或调整。

- **`LREM key count value`**: 根据参数 `count` 的值，移除列表中与参数 `value` 相等的元素。
    
- **`LSET key index value`**: 通过索引设置列表元素的值（索引必须存在）。
    
- **`LTRIM key start stop`**: 对一个列表进行修剪，只保留指定区间内的元素，不在指定区间之内的元素都将被删除。
    

---

### 5. 高级操作

- **`RPOPLPUSH source destination`**: 移除列表的最后一个元素，并将该元素添加到另一个列表并返回。
    
- **`LMOVE source destination LEFT|RIGHT LEFT|RIGHT`**: (Redis 6.2+) `RPOPLPUSH` 的更通用版本，可以自定义从哪头出，到哪头入。
    

### 常用场景建议

|**场景**|**推荐命令**|
|---|---|
|**消息队列**|`LPUSH` 生产, `RPOP` 或 `BRPOP` 消费|
|**最新消息排行榜**|`LPUSH` 插入, `LTRIM` 保持固定数量|
|**循环列表**|`RPOPLPUSH source source`|
