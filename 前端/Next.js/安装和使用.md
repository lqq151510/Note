## 环境准备

### 1. 安装 Node.js

Next.js 需要 Node.js 18.0 或更高版本。

```
# 检查当前Node.js版本
node -v

# 如果版本过低或未安装，请访问 Node.js官网 下载LTS版本
```

### 2. 选择包管理器

可以使用 npm（Node.js 自带）或 yarn：

```
# 检查npm版本
npm -v

# 安装yarn（可选）
npm install -g yarn
```

## 项目创建

### 1. 使用 Create Next App 创建项目

```
# 创建新项目
npx create-next-app@latest my-next-app

# 进入项目目录
cd my-next-app
```

### 2. 交互式配置选项

执行上述命令后，会提示以下配置选项：

- **使用 TypeScript**：`Yes`（推荐）
    
- **使用 ESLint**：`Yes`（推荐）
    
- **使用 Tailwind CSS**：`Yes`（根据需求选择）
    
- **使用 src/ 目录**：`No`（保持默认结构）
    
- **使用 App Router**：`Yes`（推荐，支持最新特性）
    
- **自定义导入别名**：`No`（可使用默认设置）
    

### 3. 安装依赖

```
# 使用npm安装依赖
npm install

# 或使用yarn
yarn install
```

### 4. 启动开发服务器

```
# 启动开发服务器
npm run dev

# 访问 http://localhost:3000 查看应用
```

## 项目结构说明

成功创建后的项目结构如下：

```
my-next-app/
├── app/                    # App Router目录（核心）
│   ├── favicon.ico        # 网站图标
│   ├── globals.css        # 全局样式文件
│   ├── layout.tsx         # 根布局组件
│   └── page.tsx          # 首页组件
├── public/                # 静态资源目录
│   ├── next.svg
│   └── vercel.svg
├── .eslintrc.json         # ESLint配置
├── .gitignore            # Git忽略文件
├── next.config.js        # Next.js配置文件
├── package.json          # 项目依赖配置
├── tailwind.config.js    # Tailwind CSS配置（如果选择）
└── tsconfig.json         # TypeScript配置（如果选择）
```

### 核心文件说明：

- **app/layout.tsx**：定义应用的根布局，包含共享组件
    
- **app/page.tsx**：首页内容，对应 `/`路由
    
- **public/**：存放静态资源，如图片、字体等
    

## 安装问题排查

### 1. 切换国内镜像源

由于网络原因，直接访问官方NPM仓库可能较慢，可以切换至国内镜像：

```
# 使用淘宝NPM镜像
npm config set registry https://registry.npmmirror.com

# 设置后重新安装依赖
npm install
```

### 2. 显示详细日志定位问题

如果安装过程卡顿，可以查看详细日志：

```
npm install --verbose
```

### 3. 清理缓存重新安装

npm缓存问题可能导致安装失败：

```
# 清理npm缓存
npm cache clean --force

# 重新安装依赖
npm install
```

### 4. 其他常见解决方案

- **权限问题**：在命令前加 `sudo`（Linux/macOS）
    
- **节点版本不匹配**：使用 nvm 管理Node.js版本
    
- **依赖冲突**：删除 node_modules 和 package-lock.json 后重新安装
