Swagger（现为**OpenAPI规范**）是一套用于**设计、构建、文档化和消费 RESTful API**​ 的开源工具集。它通过标准化格式描述API的结构，让机器和开发者都能理解接口细节。

在Java生态（特别是Spring Boot）中，常用 **SpringFox**​ 或 **SpringDoc**​ 库集成Swagger，配合注解自动生成交互式API文档。

### 常用注解（以SpringFox为例）：

#### 1. **控制器层注解**

- `@Api`：标记整个控制器类的用途(控制层上)
    
    ```
    @Api(tags = "用户管理接口")
    @RestController
    @RequestMapping("/user")
    public class UserController { ... }
    ```
    
- `@ApiOperation`：描述单个接口方法(方法上)
    
    ```
    @ApiOperation(value = "创建用户", notes = "需提供用户名和邮箱")
    @PostMapping
    public User createUser(@RequestBody User user) { ... }
    ```
    
- `@ApiParam`：描述接口参数（适用于非RequestBody参数）
    
    ```
    @GetMapping
    public User getUser(@ApiParam("用户ID") @RequestParam Long id) { ... }
    ```
    

#### 2. **模型层注解**

- `@ApiModel`：描述数据模型(类上)
    
    ```
    @ApiModel(description = "用户实体")
    public class User { ... }
    ```
    
- `@ApiModelProperty`：描述模型字段(属性上)
    
    ```
    @ApiModelProperty(value = "用户邮箱", required = true, example = "test@example.com")
    private String email;
    ```
    

#### 3. **响应注解**

- `@ApiResponse`：定义接口的响应状态码和说明
    
    ```
    @ApiResponses({
        @ApiResponse(code = 200, message = "成功"),
        @ApiResponse(code = 404, message = "用户不存在")
    })
    ```
    

---

### 注意事项：

1. **版本适配**：Spring Boot 2.6+ 推荐使用 **SpringDoc OpenAPI**（注解为 `@Operation`、`@Schema`等）
    
2. **生产环境安全**：需通过配置关闭Swagger UI的公开访问
    
3. **代码一致性**：注解描述应与实际业务逻辑保持一致
    

### 快速体验：

添加Maven依赖后访问 `http://localhost:8080/swagger-ui/`即可看到自动生成的API文档界面，支持在线调试接口。
