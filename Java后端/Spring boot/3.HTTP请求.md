HTTP请求是客户端（如浏览器）与服务器通信的基础，其格式有明确规范。一个标准的HTTP请求主要由四个部分顺序构成：**请求行**、**请求头**、**空行**和**请求体**​ 。

### 📮 HTTP请求的核心组成部分

|组成部分|核心作用与示例|详细说明|
|---|---|---|
|**请求行 (Request Line)**​|`GET /index.html HTTP/1.1`|定义了请求的**动作**（方法）、**目标**（URI）和使用的**协议版本**。这是请求的起点，告诉服务器“要做什么” 。|
|**请求头 (Request Headers)**​|`Host: www.example.com`  <br>`User-Agent: Mozilla/5.0...`  <br>`Content-Type: application/json`|以键值对的形式提供服务器所需的**附加信息**，例如客户端信息、内容类型、缓存策略等。它们就像是信的“信封信息” 。|
|**空行 (Empty Line)**​|(一个仅包含回车换行符`\r\n`的空行)|这是一个**关键分隔符**。它标志着请求头的结束。如果请求有正文，正文将紧随其后开始。没有这个空行，服务器将无法正确解析请求 。|
|**请求体 (Request Body)**​|`username=user&password=123`或 `{"name": "value"}`|包含客户端发送给服务器的**实际数据**，例如表单内容、上传的文件或API调用参数。并非所有请求都有请求体（如GET请求通常没有）。|

### 🔍 深入理解关键细节

- **请求行中的方法**：最常见的HTTP方法包括**GET**（获取资源）、**POST**（提交数据）、**PUT**（更新资源）、**DELETE**（删除资源）等。方法类型明确指出了操作意图 。
    
- **重要的请求头**：`Host`头在HTTP/1.1中**必须提供**，用于指定服务器的域名；`Content-Type`头至关重要，它声明了请求体的**数据格式**，常见的有 `application/x-www-form-urlencoded`（普通表单）、`application/json`（JSON数据）、`multipart/form-data`（带文件上传的表单）等 ；`Content-Length`头则指明了请求体的长度（字节数）。
    
- **请求体的常见格式**：根据 `Content-Type`的不同，请求体可以有多种格式，例如 `application/x-www-form-urlencoded`格式类似 `name=Alice&age=30`；`application/json`格式为 `{"name": "Alice", "age": 30}`；`multipart/form-data`格式用于混合文本和文件数据，各部分由边界字符串分隔 。
    

### 📨 请求示例

**1. GET 请求示例**

GET请求通常将参数附加在URL后，没有请求体。

```
GET /search?q=http协议 HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8
```

**2. POST 请求示例**

POST请求在请求体中提交数据。

```
POST /login HTTP/1.1
Host: www.example.com
Content-Type: application/x-www-form-urlencoded
Content-Length: 29

username=alice&password=12345
```

### 🔄 请求处理流程

下面的流程图直观展示了一个HTTP请求从构建到被服务器处理的基本过程：

```
flowchart TD
    A[客户端构建HTTP请求] --> B[发送请求]
    B --> C{服务器接收请求}
    C --> D[解析请求行<br>获取方法、URI、版本]
    D --> E[解析请求头<br>获取主机、内容类型等信息]
    E --> F[识别空行<br>确认头部结束]
    F --> G{判断是否有请求体?}
    G -- 是 --> H[根据Content-Type<br>解析请求体数据]
    G -- 否 --> I[处理请求]
    H --> I
    I --> J[生成并返回HTTP响应]
```
