Spring Boot是一款用于简化Spring应用初始搭建和开发过程的开源框架。它通过“约定优于配置”的核心原则，让开发者能快速创建独立运行、生产级别的基于Spring的应用程序。

下面的表格能帮你快速了解Spring Boot与传统Spring应用开发的主要区别：

|特性对比|Spring Boot|传统Spring应用|
|---|---|---|
|**配置方式**​|**约定优于配置**，大量自动配置|**手动配置**大量XML或注解|
|**项目依赖管理**​|**Starter**​ 依赖，一站式引入|需手动配置多个依赖及其版本|
|**部署方式**​|**内嵌Servlet容器**（如Tomcat），可打包为可执行JAR直接运行|需打包为WAR文件，部署到外部Servlet容器|
|**应用监控**​|提供**生产就绪型**的监控端点（如健康检查、指标收集）|通常需要额外集成第三方监控工具|

### 💡 核心价值：约定优于配置

Spring Boot的核心理念是“约定优于配置”（Convention over Configuration）。这意味着框架已经提供了一套合理的默认设置。例如，如果你引入了`spring-boot-starter-web`依赖，Spring Boot会自动配置Spring MVC和一个内嵌的Tomcat服务器，你通常无需再写任何配置代码就能直接启动一个Web应用。

这种做法极大地降低了决策成本，让你能更专注于业务逻辑的实现。

### 🛠️ 两大关键技术

Spring Boot通过以下两大技术实现上述理念：

- **起步依赖（Starter Dependencies）**：这些是特殊的Maven依赖描述符。每个Starter都将某个功能所需的一组相关依赖打包在一起。例如，要开发Web应用，只需引入`spring-boot-starter-web`，它会自动引入Spring MVC、Jackson（用于JSON处理）以及内嵌的Tomcat等所有相关库，避免了版本冲突和繁琐的依赖查找。
    
- **自动配置（Auto-Configuration）**：Spring Boot会根据你引入的Starter和类路径下存在的类，自动推断并配置Spring应用所需的Bean。例如，当它发现类路径下有H2数据库（一种内存数据库）的JAR包时，会自动配置一个内存数据源，让你在开发阶段开箱即用。
    

### 🚀 Spring Boot能做什么？

凭借其简洁高效的特点，Spring Boot非常适合以下场景：

- **快速构建RESTful API或传统Web应用**。
    
- **构建微服务架构**，它是Spring Cloud微服务套件的基石。
    
- **开发批处理应用程序**，通过集成Spring Batch。
    
- **构建云原生应用**，轻松适配Docker和Kubernetes等云环境。
