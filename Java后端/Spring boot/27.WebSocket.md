WebSocket 是一种网络通信协议，它通过在单个 TCP 连接上提供**全双工、双向**的通信通道，实现了客户端（如浏览器）与服务器之间的**持久性连接**和**实时数据交换**。

### 核心特点与工作原理：

1. **持久连接**：与 HTTP 的“请求-响应”模式不同，WebSocket 在初次握手建立连接后，连接会一直保持打开状态。双方可以随时主动发送数据，无需反复建立新连接。
    
2. **全双工通信**：服务器和客户端可以同时独立地发送和接收消息，就像打电话一样，沟通效率极高。
    
3. **低开销**：建立连接后，数据传输的数据包头部开销很小（通常只有2-10字节），非常适合高频、低延迟的通信场景。
    
4. **建立过程**：连接始于一个特殊的 HTTP 请求（握手），请求头中包含 `Upgrade: websocket`等信息。服务器响应“101 Switching Protocols”后，协议便从 HTTP 升级为 WebSocket，此后通信便基于 WebSocket 协议帧进行。
    

### 主要优势：

- **真正的实时性**：服务器可以主动向客户端推送数据，无需客户端轮询询问。
    
- **减少带宽和延迟**：避免了 HTTP 连接不断建立、关闭的开销和头部信息的重复传输。
    
- **更好的性能**：适用于需要快速数据更新的应用。
    

### 典型应用场景：

- 在线聊天/即时通讯
    
- 多人在线游戏
    
- 实时协作编辑（如在线文档）
    
- 股票行情、体育赛事实时比分
    
- 在线教育和视频会议的互动功能
    
- 物联网设备状态的实时监控
