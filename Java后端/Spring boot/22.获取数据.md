这三个是 Spring MVC 中用于**从 HTTP 请求中获取数据**的核心注解，它们的应用场景有明确区别。

下面是一个清晰的对比和解释：

### 核心区别速览

|注解|作用位置|常见数据格式|典型URL示例|
|---|---|---|---|
|`@RequestParam`|**URL 查询字符串**​|`key=value&key2=value2`|`/api/user?**name=张三**&**age=20**`|
|`@RequestBody`|**请求体**​|JSON, XML|（POST/PUT请求体中）`{"name":"张三", "age":20}`|
|`@PathVariable`|**URL 路径本身**​|路径中的一段|`/api/user/**123**`|

---

### 详细说明与示例

假设我们有一个管理用户（User）资源的控制器。

#### 1. `@RequestParam`- 获取查询参数

- **作用**：从URL的 `?`后面的**查询字符串**中获取参数。
    
- **场景**：用于**过滤、搜索、分页、排序**等可选或必传的条件参数。
    
- **关键属性**：
    
    - `value/name`：参数名。
        
    - `required`：是否必传，默认为 `true`（如果为 `true`但客户端没传，会报400错误）。
        
    - `defaultValue`：默认值。
        
    

```
// 示例：查找用户，支持按姓名过滤和分页
@GetMapping("/users")
public List<User> findUsers(
        @RequestParam(value = "keyword", required = false) String name, // 可选参数
        @RequestParam(defaultValue = "0") int page, // 可选，默认为0
        @RequestParam(defaultValue = "10") int size  // 可选，默认为10
) {
    // 实现查询逻辑...
}

// 对应请求：GET /users?keyword=张&page=0&size=10
```

#### 2. `@PathVariable`- 获取路径变量

- **作用**：从**URL 路径模板**中获取变量值。
    
- **场景**：用于**标识、定位一个具体的资源**（如根据ID查询、更新或删除某个用户）。
    
- **特点**：变量是**URL结构的一部分**，通常是**必选**的。
    

```
// 示例：根据用户ID获取特定用户信息
@GetMapping("/users/{id}") // {id} 是一个路径占位符
public User getUserById(@PathVariable("id") Long userId) {

    return user;
}

// 对应请求：GET /users/123
// 此时，`userId` 变量的值就是 123
```

#### 3. `@RequestBody`- 获取请求体

- **作用**：将**HTTP 请求体**中的数据（通常是JSON）绑定到方法的参数对象上。
    
- **场景**：用于**创建（POST）或更新（PUT/PATCH）**​ 一个资源，需要传递较复杂的结构化数据。
    
- **特点**：
    
    - 一个方法中**最多只能有一个**​ `@RequestBody`参数。
        
    - 通常与 `@PostMapping`或 `@PutMapping`一起使用。
        
    - **前端需要设置请求头 `Content-Type: application/json`**。
        
    

```
// 示例：创建一个新用户
@PostMapping("/users")
```

### 组合使用示例

在实际开发中，这些注解经常被组合使用。

```
// 更新指定ID的用户的部分信息
@PatchMapping("/users/{id}")
public User updateUser(
        @PathVariable Long id,          // 从路径获取要更新的用户ID
        @RequestBody Map<String, Object> updates, // 从请求体获取要更新的字段（JSON对象）
        @RequestParam(defaultValue = "false") boolean notify // 可选查询参数，是否通知用户
) {
    // 1. 根据 id 找到用户
    // 2. 根据 updates 中的键值对更新用户字段
    // 3. 如果 notify 为 true，则发送通知
    // 4. 返回更新后的用户
}

// 对应请求：
// PATCH /users/456?notify=true
// Body: {"name":"李四新名字", "age": 25}
```

### 总结选择依据

- 要**定位一个资源**（如通过ID） → 用 **`@PathVariable`**。
    
- 要传递**创建/更新资源的完整对象**​ → 用 **`@RequestBody`**。
    
- 要传递**过滤、分页等附加条件或可选参数**​ → 用 **`@RequestParam`**。
