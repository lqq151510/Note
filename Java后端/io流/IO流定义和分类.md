IO流是Java中处理输入输出的核心机制，它抽象了数据的流动，就像水流一样，数据从源头（输入）持续地“流”向目的地（输出）。

### 💡 IO流的核心作用

IO流在编程中扮演着至关重要的角色，主要体现在以下几个方面：

- **数据读写**：提供了一套标准化的方法，让程序能够轻松地从各种数据源（如文件、网络连接、键盘输入）读取数据，或将数据写入到各种目标（如文件、控制台、网络）。这简化了对不同存储介质和数据格式的操作。
    
- **设备交互**：充当程序与外部设备（如文件系统、网络套接字、甚至USB设备）进行数据交换的桥梁。这使得Java能够应用于物联网等需要设备间通信的领域。
    
- **数据缓冲**：通过缓冲区暂存数据，减少对物理设备（如硬盘）的直接读写次数。这在处理大量数据时能显著提升I/O效率。
    
- **数据转换与处理**：支持在数据传输过程中进行各种处理，例如字符编码转换、数据的加密解密、以及对象的序列化与反序列化，增强了处理复杂数据的能力。
    

### 📂 IO流的分类体系

Java的IO流体系非常庞大，但可以通过两个主要维度进行清晰划分：

|分类维度|类型|功能描述|基类/常见类|
|---|---|---|---|
|**流向**​|输入流 (Input Stream)|将数据从外部源（如文件、键盘）读取到程序内存中。|`InputStream`, `Reader`|
||输出流 (Output Stream)|将数据从程序内存写出到外部目标（如文件、屏幕）。|`OutputStream`, `Writer`|
|**操作单位**​|**字节流**​ (Byte Stream)|以**字节**为单位进行读写，能处理所有类型的数据，包括图像、音频、可执行文件等。|`InputStream`, `OutputStream`|
||**字符流**​ (Character Stream)|以**字符**为单位进行读写，专门为处理文本数据设计，能自动处理字符编码问题，避免乱码。|`Reader`, `Writer`|
|**角色与功能**​|**节点流**​ (Node Stream)|直接与数据源或目标相连，进行基础的数据读写，如 `FileInputStream`。|-|
||**处理流**​ (Processing Stream)|对已存在的流进行封装，提供增强功能（如缓冲、按行读写、数据类型转换），如 `BufferedReader`。|-|

在实际应用中，通常会组合使用这些流。例如，要高效读取一个文本文件，可以这样组合：`new BufferedReader(new InputStreamReader(new FileInputStream("file.txt"), "UTF-8"))`。这里，`FileInputStream`（字节流）是节点流，负责连接文件；`InputStreamReader`是转换流，将字节流转换为字符流；`BufferedReader`是处理流，提供缓冲功能，方便按行读取。
