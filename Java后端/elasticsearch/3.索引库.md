好的，这里是关于 **Elasticsearch 索引库（Index）**​ 的**增、删、改、查、判断**等核心操作。你可以把它理解为**数据库（Database）级别**的操作。

---

### 核心概念：什么是索引库？

在 Elasticsearch 中，**索引库**​ 是：

- 一系列具有相似特征（文档结构、映射、设置）的**文档的集合**。
    
- 类比关系型数据库：`Index`≈ `Database`。
    
- 类比 MongoDB：`Index`≈ `Collection`。
    

**注意**：Elasticsearch 7.x 之后，一个索引库默认只能包含一个**类型**，类型的概念被弱化。

---

### 1. 创建索引库

```
# 语法
PUT /索引库名称
{
  "settings": { ... }, // 索引设置，如分片数、副本数
  "mappings": { ... }  // 映射定义，即表结构
}
```

**示例**：创建一个名为 `hotel`的索引库，指定分片和映射。

```
PUT /hotel
{
  "settings": {
    "number_of_shards": 3,     // 主分片数，创建后不可修改
    "number_of_replicas": 1    // 每个主分片的副本数，可动态调整
  },
  "mappings": {
    "properties": {            // 定义字段及其类型
      "id": { "type": "long" },
      "name": { 
        "type": "text",
        "analyzer": "ik_max_word",  // 指定分词器
        "copy_to": "all"
      },
      "address": { "type": "keyword" },
      "price": { "type": "integer" },
      "brand": { "type": "keyword" },
      "city": { "type": "keyword" },
      "all": { "type": "text" }    // 用于全文搜索的复合字段
    }
  }
}
```

---

### 2. 查询索引库信息

```
# 语法
GET /索引库名称
```

**示例**：查询 `hotel`索引库的详细信息（包括设置和映射）。

```
GET /hotel
```

---

### 3. 修改索引库设置

**注意**：`settings`中的某些属性（如 `number_of_shards`）创建后**不可修改**，但 `number_of_replicas`等可以。

```
# 语法
PUT /索引库名称/_settings
{
  "number_of_replicas": 2
}
```

**示例**：修改 `hotel`索引的副本数为 2。

```
PUT /hotel/_settings
{
  "number_of_replicas": 2
}
```

---

### 4. 删除索引库

⚠️ **危险操作，不可逆！**

```
# 语法
DELETE /索引库名称
```

**示例**：删除 `hotel`索引库。

```
DELETE /hotel
```

**批量删除示例**：

```
DELETE /index1,index2
DELETE /hotel-*  // 使用通配符，删除所有以 `hotel-` 开头的索引
```

---

### 5. 判断索引库是否存在

```
# 语法
HEAD /索引库名称
```

**结果**：

- 状态码 `200`：索引库存在。
    
- 状态码 `404`：索引库不存在。
    

**示例**：

```
# 在终端中使用 curl
curl -I -X HEAD "http://localhost:9200/hotel"
```

---

### 6. 打开/关闭索引库

- **关闭索引**：索引将不再允许读写操作，但元数据保留，不占用太多计算资源。常用于归档。
    
- **打开索引**：恢复索引的正常读写。
    

```
# 关闭索引
POST /索引库名称/_close

# 打开索引
POST /索引库名称/_open
```

**示例**：

```
POST /hotel/_close
POST /hotel/_open
```

---

### 7. 常用管理命令

```
# 查看所有索引（包含健康状态、文档数、存储大小等）
GET /_cat/indices?v

# 查看索引健康状态
GET /_cat/health?v

# 查看集群节点信息
GET /_cat/nodes?v
```

---

### 最佳实践与注意事项

1. **规划分片**：`number_of_shards`创建后不可变，需根据数据量提前规划。过多分片会增加集群开销，过少会影响性能。
    
2. **使用别名**：建议为索引库创建**别名**，而不是直接在代码中写死索引名。这便于后续进行**索引重建、版本升级、零停机运维**。
    
    ```
    POST /_aliases
    {
      "actions": [
        { "add": { "index": "hotel_v2", "alias": "hotel" } }
      ]
    }
    ```
    
3. **索引模板**：对于需要按周期（如每天、每月）创建的索引（如日志 `logs-2025-01-01`），使用**索引模板**可以统一其设置和映射。
    
4. **操作前确认**：在生产环境执行 `DELETE`操作前，务必确保索引名称正确，最好有备份或快照机制。
    

**总结**：索引库是 Elasticsearch 数据组织的顶层容器。熟练掌握其创建、查询、修改和删除操作，是进行高效数据管理和应用开发的基础。核心操作是 `PUT`（创建/修改设置）、`GET`（查询）、`DELETE`（删除）和 `HEAD`（判断）。
