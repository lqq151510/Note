`RestHighLevelClient`是 Elasticsearch 官方在 7.x 版本中提供的**Java高级REST客户端**。它基于**低级客户端**构建，提供了面向对象、更易用的API，曾是Java应用连接ES的**标准官方客户端**。

---

### 1. 核心定位

- **官方出品**：由Elasticsearch官方团队维护，兼容性好。
    
- **高级封装**：将Elasticsearch的REST API封装为Java对象，简化开发。
    
- **HTTP协议**：通过HTTP/HTTPS与Elasticsearch集群通信，不依赖特定的网络协议。
    
- **当前状态**：
    
    - **已废弃**：Elasticsearch 7.15.0版本后标记为`@Deprecated`。
        
    - **被替代**：在Elasticsearch 8.x中，被新的**Java API客户端**（`elasticsearch-java`）完全取代。
        
    

> **重要**：新项目不应再使用`RestHighLevelClient`，而应使用新的Java API客户端。

---

### 2. 核心特性

```
// 核心类结构示意
RestHighLevelClient client = new RestHighLevelClient(
    RestClient.builder(
        new HttpHost("localhost", 9200, "http")
    )
);
```

- **同步/异步支持**：提供异步操作支持。
    
- **请求/响应对象**：所有操作都有对应的`Request`和`Response`对象。
    
- **兼容性**：与Elasticsearch服务端版本严格绑定，版本不匹配可能导致运行时错误。
    

---

### 3. 核心API结构

`RestHighLevelClient`将API按功能模块化，通过不同方法提供不同类型客户端：

|模块|获取方法|功能|
|---|---|---|
|**索引管理**​|`indices()`|索引的创建、删除、映射管理|
|**文档操作**​|`bulk()`, `index()`, `get()`, `update()`, `delete()`|文档的增删改查、批量操作|
|**搜索查询**​|`search()`, `scroll()`, `msearch()`|执行搜索、滚动查询、批量搜索|
|**聚合分析**​|通过`SearchRequest`实现|数据分析与统计|
|**集群管理**​|`cluster()`|集群健康、节点信息、设置管理|

### 4. 基本使用示例

#### 4.1 创建客户端

```
<!-- 旧版Maven依赖（已废弃） -->
<dependency>
    <groupId>org.elasticsearch.client</groupId>
    <artifactId>elasticsearch-rest-high-level-client</artifactId>
    <version>7.17.0</version> <!-- 与ES服务端版本一致 -->
</dependency>
```

```
// 创建客户端
RestHighLevelClient client = new RestHighLevelClient(
    RestClient.builder(
        new HttpHost("localhost", 9200, "http"),
        new HttpHost("localhost", 9201, "http")
    )
);
```

#### 4.2 索引操作示例

```
// 1. 创建索引
CreateIndexRequest request = new CreateIndexRequest("users");
request.settings(Settings.builder()
    .put("index.number_of_shards", 3)
    .put("index.number_of_replicas", 1)
);
CreateIndexResponse response = client.indices().create(request, RequestOptions.DEFAULT);

// 2. 判断索引是否存在
GetIndexRequest getRequest = new GetIndexRequest("users");
boolean exists = client.indices().exists(getRequest, RequestOptions.DEFAULT);
```

#### 4.3 文档操作示例

```
// 1. 索引文档
IndexRequest request = new IndexRequest("users");
request.id("1");
Map<String, Object> jsonMap = new HashMap<>();
jsonMap.put("name", "张三");
jsonMap.put("age", 30);
request.source(jsonMap, XContentType.JSON);
IndexResponse response = client.index(request, RequestOptions.DEFAULT);

// 2. 查询文档
GetRequest getRequest = new GetRequest("users", "1");
GetResponse getResponse = client.get(getRequest, RequestOptions.DEFAULT);

// 3. 搜索文档
SearchRequest searchRequest = new SearchRequest("users");
SearchSourceBuilder sourceBuilder = new SearchSourceBuilder();
sourceBuilder.query(QueryBuilders.matchQuery("name", "张三"));
searchRequest.source(sourceBuilder);
SearchResponse searchResponse = client.search(searchRequest, RequestOptions.DEFAULT);
```

#### 4.4 关闭客户端

```
// 必须手动关闭
client.close();
```

### 5. 迁移到新的Java API客户端

Elasticsearch 8.x引入了全新的**类型安全、流式API**的Java客户端。

#### 新旧对比

|特性|RestHighLevelClient (旧)|Java API Client (新)|
|---|---|---|
|**设计模式**​|命令式，基于Request/Response对象|声明式，基于Builder模式|
|**类型安全**​|弱类型，需手动转换JSON|强类型，自动序列化/反序列化|
|**依赖**​|与Transport Client分离|单一依赖，无传递冲突|
|**版本管理**​|版本需与ES服务端完全一致|向后兼容性更好|
|**异步支持**​|基于回调|支持CompletableFuture和反应式|

#### 迁移示例

```
<!-- 新版依赖 -->
<dependency>
    <groupId>co.elastic.clients</groupId>
    <artifactId>elasticsearch-java</artifactId>
    <version>8.13.0</version>
</dependency>
```

```
// 新版客户端创建
ElasticsearchClient client = new ElasticsearchClient(
    new RestClientTransport(
        new ApacheHttpClient(host, null, null),
        new JacksonJsonpMapper()
    )
);

// 新版搜索示例
SearchResponse<User> response = client.search(s -> s
    .index("users")
    .query(q -> q
        .match(m -> m
            .field("name")
            .query("张三")
        )
    ),
    User.class  // 自动序列化为User对象
);
```

### 6. 为什么被废弃？

1. **设计复杂**：API过于冗长，学习成本高。
    
2. **维护困难**：与低级客户端耦合，新特性同步慢。
    
3. **类型不安全**：大量使用Map和JSON字符串，容易出错。
    
4. **现代化需求**：需要更符合现代Java开发习惯的API。
    

---

### 7. 总结与建议

- **遗留系统**：如果维护使用`RestHighLevelClient`的老系统，可继续使用7.x版本，但需注意未来升级困难。
    
- **新项目**：**必须使用新的Java API客户端**（`elasticsearch-java`）。
    
- **迁移路径**：Elasticsearch官方提供了详细的迁移指南，建议逐步迁移。
    

**关键点**：`RestHighLevelClient`是一个历史产物，代表了Elasticsearch Java客户端从低级REST接口到现代化类型安全API的过渡阶段。虽然功能完整，但已被更优秀的设计取代。
