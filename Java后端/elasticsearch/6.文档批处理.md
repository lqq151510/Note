以下是 **Elasticsearch 文档批处理**​ 的详细指南，涵盖批量增删改查操作，重点介绍 **`Bulk API`**​ 的使用。

## 1. 批处理核心：Bulk API

`Bulk API`允许在一个请求中执行多个**创建、索引、更新、删除**操作，极大地提高数据处理效率。

### 基本语法格式

```
POST /_bulk
{ "action": { "metadata" } }
{ "request body" }
{ "action": { "metadata" } }
{ "request body" }
```

---

## 2. 批量操作的四种类型

|操作类型|语法|说明|
|---|---|---|
|**创建/索引**​|`{ "index": { "_index": "index", "_id": "1" } }`  <br>`{ "field": "value" }`|创建新文档，ID存在则失败/覆盖|
|**创建(仅当不存在)**​|`{ "create": { "_index": "index", "_id": "2" } }`  <br>`{ "field": "value" }`|ID不存在时才创建，存在则失败|
|**更新**​|`{ "update": { "_index": "index", "_id": "3" } }`  <br>`{ "doc": { "field": "new value" } }`|部分更新文档|
|**删除**​|`{ "delete": { "_index": "index", "_id": "4" } }`|删除文档，不需要第二行数据|

## 3. 使用 RestHighLevelClient 的批量操作

### 3.1 基础批量操作

```
public class BulkOperations {
    private RestHighLevelClient client;
    
    public void bulkMixedOperations() throws IOException {
        BulkRequest request = new BulkRequest();
        
        // 1. 索引/创建文档
        IndexRequest indexRequest1 = new IndexRequest("users")
            .id("101")
            .source(XContentType.JSON, 
                "name", "张三",
                "age", 25,
                "city", "北京");
        request.add(indexRequest1);
        
        // 2. 仅当不存在的创建
        IndexRequest createRequest = new IndexRequest("users")
            .id("102")
            .opType(DocWriteRequest.OpType.CREATE)  // 关键：CREATE操作
            .source(XContentType.JSON,
                "name", "李四",
                "age", 30,
                "city", "上海");
        request.add(createRequest);
        
        // 3. 更新文档
        Map<String, Object> updateFields = new HashMap<>();
        updateFields.put("age", 26);
        updateFields.put("update_time", "2024-01-16");
        UpdateRequest updateRequest = new UpdateRequest("users", "101")
            .doc(updateFields);
        request.add(updateRequest);
        
        // 4. 删除文档
        DeleteRequest deleteRequest = new DeleteRequest("users", "103");
        request.add(deleteRequest);
        
        // 5. 脚本更新
        Map<String, Object> params = new HashMap<>();
        params.put("increment", 1);
        Script script = new Script(ScriptType.INLINE, "painless",
            "ctx._source.age += params.increment", params);
        UpdateRequest scriptUpdate = new UpdateRequest("users", "102")
            .script(script);
        request.add(scriptUpdate);
        
        // 执行批量请求
        BulkResponse bulkResponse = client.bulk(request, RequestOptions.DEFAULT);
        
        // 处理响应
        processBulkResponse(bulkResponse);
    }
    
    private void processBulkResponse(BulkResponse response) {
        if (response.hasFailures()) {
            System.err.println("批量操作有失败项: " + response.buildFailureMessage());
        }
        
        System.out.println("批量操作耗时: " + response.getTook() + "ms");
        
        for (BulkItemResponse item : response) {
            if (item.isFailed()) {
                // 失败处理
                BulkItemResponse.Failure failure = item.getFailure();
                System.err.println("操作失败 - ID: " + item.getId() + 
                    ", 错误: " + failure.getMessage());
            } else {
                // 成功处理
                switch (item.getOpType()) {
                    case INDEX:
                    case CREATE:
                        IndexResponse indexResponse = (IndexResponse) item.getResponse();
                        System.out.println("创建/索引成功 - ID: " + indexResponse.getId() +
                            ", 结果: " + indexResponse.getResult());
                        break;
                    case UPDATE:
                        UpdateResponse updateResponse = (UpdateResponse) item.getResponse();
                        System.out.println("更新成功 - ID: " + updateResponse.getId() +
                            ", 结果: " + updateResponse.getResult());
                        break;
                    case DELETE:
                        DeleteResponse deleteResponse = (DeleteResponse) item.getResponse();
                        System.out.println("删除成功 - ID: " + deleteResponse.getId() +
                            ", 结果: " + deleteResponse.getResult());
                        break;
                }
            }
        }
    }
}
```

### 3.2 批量插入大量数据

```
public class BulkInsertExample {
    
    public void bulkInsertUsers(List<User> users) throws IOException {
        BulkRequest bulkRequest = new BulkRequest();
        
        for (User user : users) {
            IndexRequest request = new IndexRequest("users")
                .id(user.getId())
                .source(convertToMap(user), XContentType.JSON);
            bulkRequest.add(request);
        }
        
        // 可选：设置超时和刷新策略
        bulkRequest.timeout(TimeValue.timeValueMinutes(2));
        bulkRequest.setRefreshPolicy(WriteRequest.RefreshPolicy.WAIT_UNTIL);
        
        BulkResponse response = client.bulk(bulkRequest, RequestOptions.DEFAULT);
        
        if (response.hasFailures()) {
            // 处理部分失败
            handlePartialFailures(response);
        }
        
        System.out.println("批量插入完成，成功: " + 
            Arrays.stream(response.getItems())
                  .filter(item -> !item.isFailed())
                  .count() + " 条");
    }
    
    private Map<String, Object> convertToMap(User user) {
        Map<String, Object> map = new HashMap<>();
        map.put("id", user.getId());
        map.put("name", user.getName());
        map.put("age", user.getAge());
        map.put("email", user.getEmail());
        map.put("created_at", new Date());
        return map;
    }
    
    private void handlePartialFailures(BulkResponse response) {
        // 重试失败项
        BulkRequest retryRequest = new BulkRequest();
        
        for (BulkItemResponse item : response) {
            if (item.isFailed()) {
                // 重新添加失败的操作
                retryRequest.add(item.getRequest());
            }
        }
        
        if (retryRequest.numberOfActions() > 0) {
            try {
                BulkResponse retryResponse = client.bulk(retryRequest, RequestOptions.DEFAULT);
                System.out.println("重试后剩余失败: " + 
                    Arrays.stream(retryResponse.getItems())
                          .filter(BulkItemResponse::isFailed)
                          .count());
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}
```

## 5. 批量操作最佳实践

### 5.1 批量大小控制

```
public class BulkOptimizer {
    private static final int BATCH_SIZE = 1000;  // 每批处理文档数
    private static final int MAX_RETRIES = 3;     // 最大重试次数
    
    public void batchProcess(List<Map<String, Object>> documents) throws IOException {
        int total = documents.size();
        int processed = 0;
        int retryCount = 0;
        
        while (processed < total && retryCount < MAX_RETRIES) {
            int endIndex = Math.min(processed + BATCH_SIZE, total);
            List<Map<String, Object>> batch = documents.subList(processed, endIndex);
            
            if (processBatch(batch)) {
                processed = endIndex;
                retryCount = 0;  // 重置重试计数
                System.out.println("已处理: " + processed + "/" + total);
            } else {
                retryCount++;
                System.out.println("批次处理失败，重试次数: " + retryCount);
                
                if (retryCount >= MAX_RETRIES) {
                    throw new RuntimeException("批量处理失败，达到最大重试次数");
                }
                
                // 指数退避
                try {
                    Thread.sleep(1000 * (long) Math.pow(2, retryCount - 1));
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        }
    }
    
    private boolean processBatch(List<Map<String, Object>> batch) throws IOException {
        BulkRequest bulkRequest = new BulkRequest();
        
        for (Map<String, Object> doc : batch) {
            String id = (String) doc.get("id");
            IndexRequest request = new IndexRequest("batch_data")
                .id(id)
                .source(doc, XContentType.JSON);
            bulkRequest.add(request);
        }
        
        bulkRequest.timeout(TimeValue.timeValueSeconds(30));
        
        try {
            BulkResponse response = client.bulk(bulkRequest, RequestOptions.DEFAULT);
            return !response.hasFailures();
        } catch (Exception e) {
            return false;
        }
    }
}
```

### 5.2 并发批量处理

```
public class ConcurrentBulkProcessor {
    private ExecutorService executorService = Executors.newFixedThreadPool(5);
    
    public void concurrentBulkInsert(List<Map<String, Object>> allDocuments) 
            throws InterruptedException {
        int batchSize = 500;
        CountDownLatch latch = new CountDownLatch(
            (int) Math.ceil((double) allDocuments.size() / batchSize));
        
        for (int i = 0; i < allDocuments.size(); i += batchSize) {
            int end = Math.min(i + batchSize, allDocuments.size());
            List<Map<String, Object>> batch = allDocuments.subList(i, end);
            
            executorService.submit(() -> {
                try {
                    processSingleBatch(batch);
                } catch (Exception e) {
                    e.printStackTrace();
                } finally {
                    latch.countDown();
                }
            });
        }
        
        latch.await(5, TimeUnit.MINUTES);
        executorService.shutdown();
    }
    
    private void processSingleBatch(List<Map<String, Object>> batch) throws IOException {
        BulkRequest request = new BulkRequest();
        
        for (Map<String, Object> doc : batch) {
            IndexRequest indexRequest = new IndexRequest("concurrent_data")
                .source(doc, XContentType.JSON);
            request.add(indexRequest);
        }
        
        BulkResponse response = client.bulk(request, RequestOptions.DEFAULT);
        
        if (response.hasFailures()) {
            System.err.println("批次处理有失败: " + response.buildFailureMessage());
        }
    }
}
```

## 8. 性能优化建议

```
public class BulkPerformanceOptimizer {
    
    public void optimizedBulkInsert() throws IOException {
        BulkRequest request = new BulkRequest();
        
        // 1. 禁用自动刷新（大量导入时）
        request.setRefreshPolicy(WriteRequest.RefreshPolicy.NONE);
        
        // 2. 设置合适的超时时间
        request.timeout(TimeValue.timeValueMinutes(5));
        
        // 3. 禁用等待活跃分片（提高吞吐量）
        // request.waitForActiveShards(ActiveShardCount.ONE);
        
        // 4. 批量大小控制在5-15MB之间
        int totalSize = 0;
        int batchCount = 0;
        int maxBatchSize = 10 * 1024 * 1024;  // 10MB
        
        for (int i = 0; i < 10000; i++) {
            IndexRequest indexRequest = new IndexRequest("data")
                .id("doc_" + i)
                .source(createDocument(i), XContentType.JSON);
            
            // 估算文档大小
            int docSize = estimateDocumentSize(indexRequest);
            
            if (totalSize + docSize > maxBatchSize && batchCount > 0) {
                // 发送当前批次
                BulkResponse response = client.bulk(request, RequestOptions.DEFAULT);
                processResponse(response);
                
                // 重置
                request = new BulkRequest();
                totalSize = 0;
                batchCount = 0;
            }
            
            request.add(indexRequest);
            totalSize += docSize;
            batchCount++;
        }
        
        // 发送最后一批
        if (batchCount > 0) {
            BulkResponse response = client.bulk(request, RequestOptions.DEFAULT);
            processResponse(response);
        }
        
        // 5. 最后手动刷新
        RefreshRequest refreshRequest = new RefreshRequest("data");
        client.indices().refresh(refreshRequest, RequestOptions.DEFAULT);
    }
    
    private int estimateDocumentSize(IndexRequest request) {
        // 简单估算，实际应用中可能需要更精确的计算
        return request.source().length();
    }
}
```

## 9. 批量操作工具类

```
public class BulkOperationUtils {
    private RestHighLevelClient client;
    private BulkProcessor bulkProcessor;
    
    /**
     * 创建BulkProcessor（自动批处理）
     */
    public void initBulkProcessor() {
        BulkProcessor.Listener listener = new BulkProcessor.Listener() {
            @Override
            public void beforeBulk(long executionId, BulkRequest request) {
                System.out.println("开始执行批量操作，文档数: " + request.numberOfActions());
            }
            
            @Override
            public void afterBulk(long executionId, BulkRequest request, BulkResponse response) {
                if (response.hasFailures()) {
                    System.err.println("批量操作有失败: " + response.buildFailureMessage());
                } else {
                    System.out.println("批量操作成功，耗时: " + response.getTook());
                }
            }
            
            @Override
            public void afterBulk(long executionId, BulkRequest request, Throwable failure) {
                System.err.println("批量操作失败: " + failure.getMessage());
            }
        };
        
        bulkProcessor = BulkProcessor.builder(
            (request, bulkListener) -> 
                client.bulkAsync(request, RequestOptions.DEFAULT, bulkListener),
            listener)
            .setBulkActions(1000)      // 每1000个操作执行一次
            .setBulkSize(new ByteSizeValue(5, ByteSizeUnit.MB))  // 每5MB执行一次
            .setFlushInterval(TimeValue.timeValueSeconds(5))    // 每5秒执行一次
            .setConcurrentRequests(2)   // 并发请求数
            .build();
    }
    
    /**
     * 异步添加文档到批量处理器
     */
    public void addDocumentAsync(String index, String id, Map<String, Object> source) {
        IndexRequest request = new IndexRequest(index)
            .id(id)
            .source(source, XContentType.JSON);
        bulkProcessor.add(request);
    }
    
    /**
     * 关闭批量处理器
     */
    public void closeBulkProcessor() throws InterruptedException {
        if (bulkProcessor != null) {
            bulkProcessor.awaitClose(30, TimeUnit.SECONDS);
            bulkProcessor.close();
        }
    }
}
```

## 关键点总结

1. **批量大小**：建议5-15MB每批次，根据网络和服务器性能调整
    
2. **错误处理**：批量操作可能部分成功，需要处理失败项
    
3. **刷新策略**：大量导入时禁用自动刷新，导入完成后手动刷新
    
4. **并发控制**：合理设置并发数，避免压垮集群
    
5. **监控**：监控批量操作的耗时和失败率
    
6. **重试机制**：实现指数退避的重试策略
    
7. **内存管理**：控制批量请求的内存占用，避免OOM
