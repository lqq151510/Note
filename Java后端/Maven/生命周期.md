Maven 的生命周期是其构建过程的核心理念，它抽象并标准化了软件项目的构建、清理和站点生成流程。下面这个表格能帮你快速把握其整体框架：

|生命周期名称|核心目标|关键阶段示例|
|---|---|---|
|**`clean`**​|清理项目，删除之前构建生成的文件。|`clean`|
|**`default`(或 `build`)**​|完成项目的编译、测试、打包、部署等全套构建工作。|`validate`, `compile`, `test`, `package`, `install`, `deploy`|
|**`site`**​|生成项目站点文档和报告。|`site`, `site-deploy`|

### 💡 生命周期详解

**1. 三套独立的生命周期**

你需要理解的第一点是，Maven 有三套**相互独立**的生命周期：`clean`、`default`和 `site`。它们并非一个单一的流程，你可以单独调用任一生命周期的命令而不会触发其他生命周期。例如，执行 `mvn clean`只会清理项目，不会进行编译；执行 `mvn site`则只生成站点文档，不会执行打包。

**2. 阶段的有序性**

每个生命周期都由一系列**阶段**构成，这些阶段是**有序的**。当你通过命令行调用某个阶段时，Maven 会**按顺序执行该阶段及其之前的所有阶段**。

- 例如，在 `default`生命周期中执行 `mvn package`命令，Maven 会依次执行从 `validate`、`compile`到 `test`，最终执行 `package`阶段本身。
    
- 同理，执行 `mvn clean`会先执行 `pre-clean`阶段，再执行 `clean`阶段。
    

### 🛠️ 生命周期与插件的关系

Maven 的生命周期本身是**抽象**的，它仅定义了构建的各个阶段，而**具体的任务都由插件来完成**。插件是 Maven 实现各种构建功能的实际执行者。

- **插件目标**：每个插件可以提供多个功能，每个功能称为一个“目标”。例如，`maven-compiler-plugin`插件的 `compile`目标负责编译源代码。
    
- **绑定**：Maven 核心已经为生命周期的主要阶段**预绑定**了标准插件的目标。例如，`compile`阶段默认绑定了 `maven-compiler-plugin`的 `compile`目标；`package`阶段会根据项目的打包类型（如 JAR 或 WAR）绑定不同的打包插件（如 `maven-jar-plugin`）。你也可以在 POM 文件中**自定义插件绑定**，将特定插件的目标绑定到生命周期的某个阶段，从而在构建过程中执行自定义任务。
    

### 🚀 常用命令示例

理解了以上概念，这些常见的 Maven 命令就很容易掌握了：

- `mvn compile`：执行 `default`生命周期到 `compile`阶段，完成代码编译。
    
- `mvn test`：执行 `default`生命周期到 `test`阶段，运行单元测试。
    
- `mvn package`：执行完整的编译、测试流程，并将代码打包。
    
- `mvn clean install`：这是一个组合命令。先执行 `clean`生命周期中的 `clean`阶段以清理旧构建，再执行 `default`生命周期到 `install`阶段，完成构建后将产物安装到本地仓库。这是开发中非常常用的命令。
    
- `mvn clean clean`：移除上一次构建生成的文件。
