Nacos 作为注册中心，其搭建和使用可以概括为以下核心步骤和原理。这里以 Spring Cloud Alibaba 微服务项目为例进行说明。

### 一、Nacos Server 搭建

1. **下载与启动**：
    
    - 从 Nacos 官网下载最新稳定版。
        
    - **单机模式**：进入 `bin`目录，执行 `startup.cmd -m standalone`(Windows) 或 `sh startup.sh -m standalone`(Linux/macOS)。
        
    - **集群模式**：需配置 `cluster.conf`和数据库，详情请参考官方文档。
        
    
2. **访问控制台**：
    
    - 启动后，访问 `http://localhost:8848/nacos`。
        
    - 默认账号密码均为 `nacos`。
        
    

### 二、服务注册（Service Provider）

服务提供者需要将自己的信息注册到 Nacos。

1. **添加依赖**：在项目的 `pom.xml`中引入 Spring Cloud Alibaba Nacos Discovery 依赖。
    
    ```
    <dependency>
        <groupId>com.alibaba.cloud</groupId>
        <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    </dependency>
    ```
    
2. **配置 Nacos 地址**：在 `application.yml`中配置。
    
    ```
    spring:
      application:
        name: user-service # 服务名称，用于服务发现的关键标识
      cloud:
        nacos:
          discovery:
            server-addr: localhost:8848 # Nacos Server 地址
    ```
    
3. **启用服务发现**：在主应用类上添加 `@EnableDiscoveryClient`注解。
    
4. **启动服务**：启动应用后，即可在 Nacos 控制台的 **“服务管理” -> “服务列表”**​ 中看到名为 `user-service`的服务及其实例。
    

### 三、服务发现与调用（Service Consumer）

服务消费者从 Nacos 获取提供者实例列表并进行调用。

1. **相同配置**：同样需要添加依赖、配置 Nacos 地址、启用服务发现（步骤同上）。
    
2. **使用负载均衡器调用**：Spring Cloud 默认整合了 Ribbon 或 Spring Cloud LoadBalancer 进行客户端负载均衡。
    
    ```
    @RestController
    public class OrderController {
        // 使用 LoadBalanced 注解使 RestTemplate 具备负载均衡能力
        @Bean
        @LoadBalanced
        public RestTemplate restTemplate() {
            return new RestTemplate();
        }
    
        @Autowired
        private RestTemplate restTemplate;
    
        public String createOrder() {
            // 直接使用服务名进行调用，而非具体的IP:PORT
            String url = "http://user-service/user/info/1";
            return restTemplate.getForObject(url, String.class);
        }
    }
    ```
    
    - 也可以使用 `OpenFeign`（声明式HTTP客户端）进行调用，更为简洁。
        
    

### 四、负载均衡

负载均衡在**服务消费者侧**自动完成，是服务发现的核心环节。

- **原理**：当消费者通过服务名（如 `user-service`）发起调用时：
    
    1. 负载均衡器（如 Ribbon）会从 Nacos 获取该服务所有健康实例的列表。
        
    2. 根据内置的**负载均衡策略**（默认为轮询）选择一个实例。
        
    3. 将请求转发到该实例的 `IP:PORT`。
        
    
- **常见策略**：轮询、随机、根据响应时间加权等。可通过配置更改。
    

### 总结流程

1. **搭建**：启动 Nacos Server。
    
2. **注册**：Provider 启动后，自动向 Nacos 注册。
    
3. **发现**：Consumer 调用时，向 Nacos 查询 Provider 实例列表。
    
4. **负载均衡**：Consumer 的负载均衡器从列表中选择一个实例发起调用。
    

**注意**：请确保 Spring Boot、Spring Cloud 和 Spring Cloud Alibaba 的版本兼容。具体依赖版本请参考官方版本说明文档。
