**OpenFeign 连接池**​ 是通过替换默认的 HTTP 客户端（JDK 自带的 `HttpURLConnection`）来实现的。默认的 HTTP 客户端效率低、不支持连接池，而使用 **Apache HttpClient**​ 或 **OKHttp**​ 可以提供连接池、长连接复用、更高效的 I/O 等能力，显著提高性能。

以下是配置 OpenFeign 使用连接池的完整指南：

## 一、Apache HttpClient 连接池（推荐）

Apache HttpClient 功能全面，是生产环境中最常用的选择。

### 1. 添加依赖

```
<!-- 在服务消费者的 pom.xml 中添加 -->
<dependency>
    <groupId>io.github.openfeign</groupId>
    <artifactId>feign-httpclient</artifactId>
</dependency>

<!-- 如果使用的是 Spring Cloud 2020+，需添加 loadbalancer -->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>
```

### 2. 启用连接池配置

在 `application.yml`中配置：

```
feign:
  httpclient:
    enabled: true  # 启用 Apache HttpClient，默认为 false
    max-connections: 200  # 最大连接数，默认 200
    max-connections-per-route: 50  # 每个路由的最大连接数，默认 50
    connection-timeout: 2000  # 连接超时时间（ms），默认 2000
    time-to-live: 900  # 连接存活时间（秒），默认 900
    follow-redirects: true  # 是否跟随重定向，默认 true
    disable-ssl-validation: false  # 是否禁用 SSL 验证，默认 false
```

### 3. 自定义连接池配置

如果需要更细粒度的控制，可以创建配置类：

```
import org.apache.http.client.HttpClient;
import org.apache.http.impl.client.HttpClientBuilder;
import org.apache.http.impl.conn.PoolingHttpClientConnectionManager;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class FeignClientConfig {
    
    @Bean
    public HttpClient httpClient() {
        // 创建连接池管理器
        PoolingHttpClientConnectionManager connectionManager = 
            new PoolingHttpClientConnectionManager();
        
        // 设置整个连接池最大连接数
        connectionManager.setMaxTotal(200);
        
        // 设置每个路由（目标主机）的最大连接数
        connectionManager.setDefaultMaxPerRoute(50);
        
        // 设置连接存活时间（单位：秒）
        connectionManager.setValidateAfterInactivity(30000);
        
        return HttpClientBuilder.create()
                .setConnectionManager(connectionManager)
                .evictExpiredConnections()  // 驱逐过期连接
                .evictIdleConnections(30, TimeUnit.SECONDS)  // 驱逐空闲连接
                .build();
    }
}
```

## 二、OKHttp 连接池

OKHttp 是 Square 开发的现代 HTTP 客户端，性能优异，支持 HTTP/2。

### 1. 添加依赖

```
<dependency>
    <groupId>io.github.openfeign</groupId>
    <artifactId>feign-okhttp</artifactId>
</dependency>
```

### 2. 启用 OKHttp

```
feign:
  okhttp:
    enabled: true
  # OKHttp 特定配置
  client:
    config:
      default:
        connectTimeout: 5000
        readTimeout: 10000
        loggerLevel: basic
```

### 3. 自定义 OKHttp 客户端

```
import okhttp3.ConnectionPool;
import okhttp3.OkHttpClient;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import java.util.concurrent.TimeUnit;

@Configuration
public class OkHttpConfig {
    
    @Bean
    public OkHttpClient okHttpClient() {
        return new OkHttpClient.Builder()
                .connectTimeout(5, TimeUnit.SECONDS)  // 连接超时
                .readTimeout(10, TimeUnit.SECONDS)    // 读取超时
                .writeTimeout(10, TimeUnit.SECONDS)   // 写入超时
                
                // 连接池配置
                .connectionPool(new ConnectionPool(
                    200,  // 最大空闲连接数
                    5,    // 连接保持时间（分钟）
                    TimeUnit.MINUTES
                ))
                
                // 重试
                .retryOnConnectionFailure(true)
                
                .build();
    }
}
```

## 三、连接池参数详解

### 核心参数含义：

|参数|说明|建议值|
|---|---|---|
|**max-connections**​|整个连接池的最大连接数|200-1000（根据机器配置调整）|
|**max-connections-per-route**​|每个目标主机的最大连接数|20-100|
|**connection-timeout**​|建立连接的超时时间|2000-5000ms|
|**time-to-live**​|连接的最大存活时间|5-15分钟（300-900秒）|
|**validateAfterInactivity**​|连接空闲多长时间后验证有效性|30000ms|

## 四、连接池监控

### 1. 添加监控端点

```
management:
  endpoints:
    web:
      exposure:
        include: "metrics,health,httptrace"
  metrics:
    export:
      prometheus:
        enabled: true
```

### 2. 关键指标

连接池的健康状态可以通过以下指标监控：

- `httpclient.total.connections`：总连接数
    
- `httpclient.idle.connections`：空闲连接数
    
- `httpclient.pending.connections`：等待连接的请求数
    
- 请求成功率、失败率、超时率
    

## 五、最佳实践

### 1. 合理设置连接池大小

```
// 计算公式参考
int maxConnections = Runtime.getRuntime().availableProcessors() * 50;
int maxPerRoute = Runtime.getRuntime().availableProcessors() * 10;
```

### 2. 超时设置策略

```
feign:
  client:
    config:
      default:
        connectTimeout: 3000  # 连接超时
        readTimeout: 10000    # 读取超时
        loggerLevel: basic
  httpclient:
    enabled: true
    connection-timeout: 3000
```

### 3. 结合 Sentinel 进行流控

```
@Configuration
public class SentinelConfig {
    
    @Bean
    @Scope("prototype")
    public Feign.Builder feignSentinelBuilder() {
        return Feign.builder()
                .client(new SentinelFeignClient(new Client.Default(null, null)));
    }
}
```

## 六、常见问题

### 1. 连接泄露

**现象**：连接数持续增长，不释放。

**解决**：

- 检查是否设置了合理的 `time-to-live`
    
- 确认是否有未关闭的响应体
    
- 使用连接监控工具
    

### 2. 连接拒绝

**现象**：`Connection refused`或连接超时。

**解决**：

- 增加 `max-connections`
    
- 检查目标服务是否健康
    
- 调整超时时间
    

### 3. 性能调优

```
# 生产环境推荐配置
feign:
  httpclient:
    enabled: true
    max-connections: 500
    max-connections-per-route: 100
    connection-timeout: 3000
    time-to-live: 300
    connection-timer-repeat: 3000
  client:
    config:
      default:
        connectTimeout: 5000
        readTimeout: 15000
        loggerLevel: basic
```

## 七、验证连接池是否生效

### 1. 查看启动日志

启动时查看日志，如果有以下输出，说明连接池已启用：

```
2026-XX-XX XX:XX:XX.XXX INFO  [main] o.s.c.o.c.h.HttpClientFeignLoadBalancedConfiguration : 
Feign HTTP Client is using Apache HttpClient.
```

### 2. 查看连接池状态

通过 JMX 或 Actuator 端点查看连接池状态：

```
GET /actuator/metrics/httpclient.total.connections
```

### 3. 性能测试

使用压测工具（如 JMeter）对比启用连接池前后的 QPS 和响应时间。

## 总结

1. **必选步骤**：添加 `feign-httpclient`或 `feign-okhttp`依赖
    
2. **关键配置**：`feign.httpclient.enabled=true`
    
3. **调优重点**：`max-connections`和 `max-connections-per-route`
    
4. **监控必备**：通过 Actuator 监控连接池状态
    

**建议**：对于大多数生产场景，推荐使用 **Apache HttpClient**，它更稳定、功能更全面。对于移动端或对 HTTP/2 有强烈需求的场景，可以选择 **OKHttp**。
