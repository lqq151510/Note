# RabbitMQ Fanout 交换机

Fanout（扇形）交换机是 RabbitMQ 中最简单的交换机类型，它将**接收到的所有消息广播到所有绑定的队列**，实现**发布/订阅**模式。

## 一、 核心特性

### 1. 工作模式

```
[Fanout Exchange]
              │
      ┌───────┼───────┐
      ▼       ▼       ▼
  [Queue1] [Queue2] [Queue3]
      │       │       │
      ▼       ▼       ▼
  [Consumer1][Consumer2][Consumer3]
```

- **广播机制**：消息发送到 Fanout 交换机时，**忽略 Routing Key**
    
- **多队列接收**：交换机会将消息**复制多份**，分别发送到所有绑定的队列
    
- **无选择性**：所有绑定队列都会收到完全相同的消息副本
    

### 2. 适用场景

- 实时通知系统（如订单状态变更通知多系统）
    
- 日志收集与分发
    
- 事件总线
    
- 实时数据同步
    
- 广播消息
    

## 二、 基础实现示例

### 1. 生产者

```
import com.rabbitmq.client.Channel;
import com.rabbitmq.client.Connection;
import com.rabbitmq.client.ConnectionFactory;
import java.nio.charset.StandardCharsets;

public class FanoutProducer {
    
    private static final String EXCHANGE_NAME = "fanout_exchange";
    
    public static void main(String[] args) throws Exception {
        ConnectionFactory factory = new ConnectionFactory();
        factory.setHost("localhost");
        
        try (Connection connection = factory.newConnection();
             Channel channel = connection.createChannel()) {
            
            // 1. 声明 Fanout 交换机
            // 参数：交换机名，交换机类型，是否持久化
            channel.exchangeDeclare(EXCHANGE_NAME, "fanout", true);
            
            // 发送消息
            for (int i = 1; i <= 5; i++) {
                String message = String.format("广播消息#%d: %s", i, 
                    java.time.LocalDateTime.now().toString());
                
                // 2. 发送到 Fanout 交换机（Routing Key 被忽略）
                // 注意：第二个参数（routingKey）可以为空或任意值
                channel.basicPublish(EXCHANGE_NAME, "", null, 
                    message.getBytes(StandardCharsets.UTF_8));
                
                System.out.println(" [x] 广播: '" + message + "'");
                Thread.sleep(1000);
            }
        }
    }
}
```

### 2. 消费者

```
import com.rabbitmq.client.*;
import java.nio.charset.StandardCharsets;

public class FanoutConsumer {
    
    private static final String EXCHANGE_NAME = "fanout_exchange";
    private final String consumerName;
    private String queueName; // 随机队列名
    
    public FanoutConsumer(String name) {
        this.consumerName = name;
    }
    
    public void start() throws Exception {
        ConnectionFactory factory = new ConnectionFactory();
        factory.setHost("localhost");
        
        Connection connection = factory.newConnection();
        Channel channel = connection.createChannel();
        
        // 1. 声明 Fanout 交换机（必须与生产者一致）
        channel.exchangeDeclare(EXCHANGE_NAME, "fanout", true);
        
        // 2. 创建临时队列
        // 参数：不指定队列名（RabbitMQ生成随机名），非持久，排他，自动删除
        queueName = channel.queueDeclare().getQueue();
        System.out.println(consumerName + " 队列名: " + queueName);
        
        // 3. 绑定队列到交换机
        // 注意：Fanout 交换机忽略 routingKey，可以为空字符串
        channel.queueBind(queueName, EXCHANGE_NAME, "");
        
        System.out.println(consumerName + " 等待广播消息...");
        
        // 4. 创建消费者回调
        DeliverCallback deliverCallback = (consumerTag, delivery) -> {
            String message = new String(delivery.getBody(), StandardCharsets.UTF_8);
            System.out.println(consumerName + " 收到: '" + message + "'");
        };
        
        // 5. 开始消费
        channel.basicConsume(queueName, true, deliverCallback, consumerTag -> {});
        
        // 保持消费者运行
        Thread.sleep(30000);
        connection.close();
    }
    
    public static void main(String[] args) throws Exception {
        // 启动多个消费者测试广播
        new FanoutConsumer("消费者A").start();
        new FanoutConsumer("消费者B").start();
        new FanoutConsumer("消费者C").start();
    }
}
```

```
@Configuration
public class FanoutConfig {
    
    // Fanout 交换机
    @Bean
    public FanoutExchange fanoutExchange() {
        return ExchangeBuilder.fanoutExchange("spring.fanout.exchange")
                .durable(true)  // 持久化
                .build();
    }

    public Queue emailQueue() {

                .withArgument("x-max-length", 10000) // 队列最大长度
                .build();
    }
    
    // 队列2 - 用于短信服务
    @Bean
    public Queue smsQueue() {
        return QueueBuilder.durable("fanout.queue.sms")
                .withArgument("x-message-ttl", 60000) // 消息TTL 60秒
                .build();
    }
    
    // 队列3 - 用于日志服务
    @Bean
    public Queue logQueue() {
        return QueueBuilder.durable("fanout.queue.log")
                .build();

    // 队列4 - 临时队列（非持久化，自动删除）

    public Queue tempQueue() {
        return new Queue("fanout.queue.temp", false, true, true);
```

```
public class FanoutProducer {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    @Autowired
    private FanoutExchange fanoutExchange;

     * 发送广播消息
     * @param message 消息内容
     */
    public void broadcast(String message) {
        // 发送到 Fanout 交换机，routingKey 被忽略
        rabbitTemplate.convertAndSend(fanoutExchange.getName(), "", message);
        System.out.println("广播消息: " + message);
    }
    
    /**
     * 发送订单创建事件
     */
    public void sendOrderCreatedEvent(OrderEvent event) {
        String message = String.format("订单创建事件 - 订单号: %s, 金额: %.2f, 时间: %s",
                event.getOrderId(), event.getAmount(), event.getCreateTime());
        
        rabbitTemplate.convertAndSend(fanoutExchange.getName(), "", message);
        
        // 也可以发送对象
        rabbitTemplate.convertAndSend(fanoutExchange.getName(), "", event);
    }
    
    /**
     * 发送系统通知
     */
    public void sendSystemNotification(String title, String content) {
        Map<String, Object> notification = new HashMap<>();
        notification.put("type", "SYSTEM_NOTIFICATION");
        notification.put("title", title);
        notification.put("content", content);
        notification.put("timestamp", System.currentTimeMillis());
        
        rabbitTemplate.convertAndSend(fanoutExchange.getName(), "", notification);
    }
    
    // 事件对象
    @Data
    @NoArgsConstructor
    @AllArgsConstructor
    public static class OrderEvent {
        private String orderId;
        private BigDecimal amount;
        private LocalDateTime createTime;
    }
}
```

## 四、 动态绑定与解绑

### 1. 动态绑定队列

```
@Component
public class DynamicBindingService {
    
    @Autowired
    private AmqpAdmin amqpAdmin;
    
    @Autowired
    private FanoutExchange fanoutExchange;
    
    /**
     * 动态绑定新队列到 Fanout 交换机
     */
    public void bindNewQueue(String queueName) {
        // 创建队列
        Queue queue = new Queue(queueName, true);
        amqpAdmin.declareQueue(queue);
        
        // 绑定到 Fanout 交换机
        Binding binding = new Binding(
            queueName,
            Binding.DestinationType.QUEUE,
            fanoutExchange.getName(),
            "", // Fanout 交换机的 routingKey 为空
            null
        );
        
        amqpAdmin.declareBinding(binding);
        System.out.println("队列 " + queueName + " 已绑定到 Fanout 交换机");
    }
    
    /**
     * 动态解绑队列
     */
    public void unbindQueue(String queueName) {
        Binding binding = new Binding(
            queueName,
            Binding.DestinationType.QUEUE,
            fanoutExchange.getName(),
            "",
            null
        );
        
        amqpAdmin.removeBinding(binding);
        System.out.println("队列 " + queueName + " 已从 Fanout 交换机解绑");
    }
    
    /**
     * 创建临时队列并绑定
     */
    public String createTempQueueAndBind() {
        // 创建临时队列（非持久化，排他，自动删除）
        Queue queue = new Queue("", false, true, true);
        String queueName = amqpAdmin.declareQueue(queue);
        
        // 绑定
        Binding binding = BindingBuilder.bind(queue).to(fanoutExchange);
        amqpAdmin.declareBinding(binding);
        
        return queueName;
    }
}
```

### 2. 动态消费者

```
@Component
public class DynamicConsumer {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    private final Map<String, String> consumerTags = new ConcurrentHashMap<>();
    
    /**
     * 动态创建消费者
     */
    public void createDynamicConsumer(String queueName) {
        String consumerTag = UUID.randomUUID().toString();
        
        // 手动创建消费者
        rabbitTemplate.execute(channel -> {
            DefaultConsumer consumer = new DefaultConsumer(channel) {
                @Override
                public void handleDelivery(String consumerTag, Envelope envelope,
                                         AMQP.BasicProperties properties, byte[] body) {
                    String message = new String(body, StandardCharsets.UTF_8);
                    System.out.println("[动态消费者-" + queueName + "] 收到: " + message);
                    
                    try {
                        channel.basicAck(envelope.getDeliveryTag(), false);
                    } catch (IOException e) {
                        e.printStackTrace();
                    }
                }
            };
            
            String actualTag = channel.basicConsume(queueName, false, consumer);
            consumerTags.put(queueName, actualTag);
            return null;
        });
        
        System.out.println("动态消费者已创建，队列: " + queueName);
    }
    
    /**
     * 取消消费者
     */
    public void cancelConsumer(String queueName) {
        String consumerTag = consumerTags.get(queueName);
        if (consumerTag != null) {
            rabbitTemplate.execute(channel -> {
                channel.basicCancel(consumerTag);
                consumerTags.remove(queueName);
                return null;
            });
            System.out.println("消费者已取消，队列: " + queueName);
        }
    }
}
```

```
public class FanoutConsumerConfig {

    public SimpleRabbitListenerContainerFactory fanoutListenerFactory(

        SimpleRabbitListenerContainerFactory factory = new SimpleRabbitListenerContainerFactory();
        factory.setConnectionFactory(connectionFactory);

        return factory;
    }
```

### 场景2：实时日志收集

```
@Component
public class LogBroadcastService {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    @Autowired
    private FanoutExchange logExchange;
    
    public void broadcastLog(String level, String message, String service) {
        Map<String, Object> logEntry = new HashMap<>();
        logEntry.put("level", level);
        logEntry.put("message", message);
        logEntry.put("service", service);
        logEntry.put("timestamp", System.currentTimeMillis());
        logEntry.put("host", getHostName());
        
        rabbitTemplate.convertAndSend(logExchange.getName(), "", logEntry);
    }
    
    public void broadcastError(Exception e, String context) {
        Map<String, Object> errorLog = new HashMap<>();
        errorLog.put("type", "ERROR");
        errorLog.put("exception", e.getClass().getName());
        errorLog.put("message", e.getMessage());
        errorLog.put("context", context);
        errorLog.put("stackTrace", Arrays.toString(e.getStackTrace()));
        
        rabbitTemplate.convertAndSend(logExchange.getName(), "", errorLog);
    }
    
    private String getHostName() {
        try {
            return InetAddress.getLocalHost().getHostName();
        } catch (UnknownHostException e) {
            return "unknown";
        }
    }
}
```

```
public class FanoutMonitor {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    /**
     * 获取交换机统计信息
     */
    public Map<String, Object> getExchangeStats(String exchangeName) {
        return rabbitTemplate.execute(channel -> {
            AMQP.Exchange.DeclareOk declareOk = channel.exchangeDeclarePassive(exchangeName);
            
            Map<String, Object> stats = new HashMap<>();
            stats.put("exchange", exchangeName);
            stats.put("type", "fanout");
            stats.put("durable", true);
            stats.put("bindings", getBindingsCount(channel, exchangeName));
            
            return stats;
        });
    }
    
    private int getBindingsCount(Channel channel, String exchangeName) throws IOException {
        AMQP.Queue.BindOk[] bindings = channel.queueBindings(exchangeName);
        return bindings != null ? bindings.length : 0;
    }
}
```

## 八、 最佳实践

1. **使用场景**：Fanout 交换机适用于真正的广播场景，所有消费者都需要相同的消息
    
2. **队列管理**：
    
    - 为不同的消费者组创建独立的队列
        
    - 临时队列设置自动删除
        
    - 重要队列设置持久化
        
    
3. **性能考虑**：
    
    - Fanout 交换机会复制消息，增加网络开销
        
    - 绑定大量队列时注意性能影响
        
    
4. **消息设计**：
    
    - 消息应包含足够的信息供不同消费者使用
        
    - 考虑添加消息类型字段供消费者过滤
        
    
5. **错误处理**：
    
    - 重要的广播消息应持久化
        
    - 为关键业务队列配置死信队列
        
    
6. **监控告警**：
    
    - 监控队列长度
        
    - 设置消费者离线告警
        
    

## 九、 常见问题

### Q1: Fanout 和 Direct 交换机的区别？

- **Fanout**：广播到所有绑定队列，忽略 Routing Key
    
- **Direct**：精确匹配 Routing Key，路由到特定队列
    

### Q2: 如何实现选择性广播？

- 方案1：使用多个 Fanout 交换机
    
- 方案2：使用 Topic 交换机配合通配符
    
- 方案3：消费者端过滤消息
    

### Q3: 临时队列断开后消息会丢失吗？

- 会。临时队列（非持久化，自动删除）在消费者断开连接后会被删除
    
- 重要消息应使用持久化队列
    

### Q4: 如何避免消息重复处理？

- 实现消费者幂等性
    
- 消息添加唯一ID
    
- 记录已处理的消息ID
    

Fanout 交换机是构建发布/订阅系统的理想选择，特别适合需要将同一消息广播给多个独立系统的场景。
