# 基于注解声明队列和交换机（Spring Boot）

注解声明是 Spring Boot 中更简洁的 RabbitMQ 配置方式，特别适合**代码集中、配置简单、快速开发**的场景。

## 一、 基础注解声明

### 1. 在 `@RabbitListener`中声明

```
@Component
public class AnnotationConsumer {
    
    /**
     * 基本声明 - Direct
     */
    @RabbitListener(bindings = @QueueBinding(
        value = @Queue(value = "direct.queue", durable = "true"),
        exchange = @Exchange(value = "direct.exchange", type = ExchangeTypes.DIRECT),
        key = "direct.key"
    ))
    public void handleDirect(String message) {
        System.out.println("Direct消息: " + message);
    }
    
    /**
     * 基本声明 - Topic
     */
    @RabbitListener(bindings = @QueueBinding(
        value = @Queue("topic.queue"),
        exchange = @Exchange(value = "topic.exchange", type = ExchangeTypes.TOPIC),
        key = "topic.*.key"
    ))
    public void handleTopic(String message) {
        System.out.println("Topic消息: " + message);
    }
    
    /**
     * 基本声明 - Fanout
     */
    @RabbitListener(bindings = @QueueBinding(
        value = @Queue("fanout.queue"),
        exchange = @Exchange(value = "fanout.exchange", type = ExchangeTypes.FANOUT)
        // Fanout 不需要 key
    ))
    public void handleFanout(String message) {
        System.out.println("Fanout消息: " + message);
    }
    
    /**
     * 基本声明 - Headers
     */
    @RabbitListener(bindings = @QueueBinding(
        value = @Queue("headers.queue"),
        exchange = @Exchange(value = "headers.exchange", type = ExchangeTypes.HEADERS),
        arguments = {
            @Argument(name = "x-match", value = "all"),
            @Argument(name = "type", value = "user")
        }
    ))
    public void handleHeaders(String message) {
        System.out.println("Headers消息: " + message);
    }
}
```

### 2. 延迟交换机（插件）

```
@Component
public class DelayedExchangeAnnotation {
    
    /**
     * 延迟交换机（需要 rabbitmq_delayed_message_exchange 插件）
     */
    @RabbitListener(bindings = @QueueBinding(
        value = @Queue(value = "delayed.queue", durable = "true"),
        exchange = @Exchange(
            value = "delayed.exchange",
            type = "x-delayed-message",  // 自定义类型
            durable = "true",
            arguments = {
                @Argument(name = "x-delayed-type", value = "direct")
            }
        ),
        key = "delayed.key"
    ))
    public void handleDelayedMessage(String message,
                                    @Header(required = false, name = "x-delay") Integer delay) {
        System.out.println("延迟消息 [延迟:" + delay + "ms]: " + message);
    }
    
    /**
     * 生产者示例
     */
    @Component
    public static class DelayedProducer {
        
        @Autowired
        private RabbitTemplate rabbitTemplate;
        
        public void sendDelayedMessage(String message, int delayMillis) {
            // 设置延迟头
            MessagePostProcessor processor = message1 -> {
                message1.getMessageProperties().setHeader("x-delay", delayMillis);
                return message1;
            };
            
            rabbitTemplate.convertAndSend("delayed.exchange", "delayed.key", 
                                         message, processor);
            System.out.println("发送延迟消息: " + delayMillis + "ms 后投递");
        }
    }
}
```

### 2. 自定义容器工厂配置

```
@Configuration
public class ContainerFactoryConfig {
    
    @Bean
    public SimpleRabbitListenerContainerFactory customContainerFactory(
            ConnectionFactory connectionFactory) {
        
        SimpleRabbitListenerContainerFactory factory = new SimpleRabbitListenerContainerFactory();
        factory.setConnectionFactory(connectionFactory);
        
        // 并发配置
        factory.setConcurrentConsumers(3);
        factory.setMaxConcurrentConsumers(10);
        
        // 确认模式
        factory.setAcknowledgeMode(AcknowledgeMode.MANUAL);
        
        // 预取数量
        factory.setPrefetchCount(1);
        
        // 重试配置
        RetryTemplate retryTemplate = new RetryTemplate();
        ExponentialBackOffPolicy backOffPolicy = new ExponentialBackOffPolicy();
        backOffPolicy.setInitialInterval(1000);
        backOffPolicy.setMultiplier(2.0);
        backOffPolicy.setMaxInterval(10000);
        
        SimpleRetryPolicy retryPolicy = new SimpleRetryPolicy();
        retryPolicy.setMaxAttempts(3);
        
        retryTemplate.setBackOffPolicy(backOffPolicy);
        retryTemplate.setRetryPolicy(retryPolicy);
        
        factory.setRetryTemplate(retryTemplate);
        
        // 错误处理器
        factory.setErrorHandler(new ConditionalRejectingErrorHandler(
            new CustomExceptionStrategy()
        ));
        
        // 消息转换器
        factory.setMessageConverter(new Jackson2JsonMessageConverter());
        
        return factory;
    }
    
    // 自定义异常策略
    public static class CustomExceptionStrategy extends ConditionalRejectingErrorHandler.DefaultExceptionStrategy {
        @Override
        public boolean isFatal(Throwable t) {
            return t instanceof MessageConversionException || 
                   t instanceof org.springframework.amqp.AmqpRejectAndDontRequeueException;
        }
    }
}
```

## 十、 常见问题

### Q1: 注解声明 vs Bean 声明？

- **注解声明**：代码集中，适合简单配置，开发速度快
    
- **Bean 声明**：配置灵活，支持复杂场景，易于管理
    

### Q2: 注解声明的执行顺序？

1. `@RabbitListener`注解解析
    
2. 队列、交换机、绑定声明
    
3. 监听器容器创建
    
4. 消息消费
    

### Q3: 如何避免重复声明？

- 使用 `id`属性
    
- 统一命名规范
    
- 配置中心管理
    

### Q4: 注解声明的限制？

- 不支持动态参数
    
- 复杂配置不够灵活
    
- 调试相对困难
    

注解声明提供了简洁、集中的配置方式，特别适合中小型项目和快速原型开发。通过合理使用各种注解属性，可以实现大部分 RabbitMQ 功能需求。
