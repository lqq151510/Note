# RabbitMQ 工作队列（Work Queues）

工作队列（也称为任务队列）是 RabbitMQ 的核心模式之一，主要用于在多个消费者之间分配**耗时任务**，实现**负载均衡**。

## 一、 工作队列的核心概念

### 1. 主要目的

- **任务分发**：将耗时任务分配给多个工作进程
    
- **负载均衡**：避免单个消费者过载
    
- **解耦**：生产者无需等待任务完成
    
- **横向扩展**：轻松增加消费者提高处理能力
    

### 2. 消息分发策略

RabbitMQ 默认使用 **循环分发（Round-robin）**​ 策略：

```
生产者 → [消息1, 消息2, 消息3, 消息4, 消息5] → 队列
                        ↓
消费者A: 处理消息1、消息3、消息5
消费者B: 处理消息2、消息4
```

## 二、 基础实现示例

### 1. 生产者（任务发送方）

```
import com.rabbitmq.client.Channel;
import com.rabbitmq.client.Connection;
import com.rabbitmq.client.ConnectionFactory;
import java.nio.charset.StandardCharsets;

public class TaskProducer {
    
    private static final String QUEUE_NAME = "work_queue";
    
    public static void main(String[] args) throws Exception {
        // 创建连接工厂
        ConnectionFactory factory = new ConnectionFactory();
        factory.setHost("localhost");
        
        try (Connection connection = factory.newConnection();
             Channel channel = connection.createChannel()) {
            
            // 声明队列（持久化）
            boolean durable = true; // 队列持久化
            channel.queueDeclare(QUEUE_NAME, durable, false, false, null);
            
            // 发送10个任务
            for (int i = 1; i <= 10; i++) {
                String task = "任务#" + i;
                String message = String.format("Task %d: 处理任务需要 %d 秒", i, i);
                
                // 设置消息持久化
                channel.basicPublish("", QUEUE_NAME, 
                    MessageProperties.PERSISTENT_TEXT_PLAIN,
                    message.getBytes(StandardCharsets.UTF_8));
                
                System.out.println(" [x] 发送: '" + task + "'");
                
                // 模拟任务间隔
                Thread.sleep(500);
            }
        }
    }
}
```

### 2. 消费者（工作进程）

```
import com.rabbitmq.client.*;
import java.nio.charset.StandardCharsets;

public class Worker {
    
    private static final String QUEUE_NAME = "work_queue";
    private final String workerName;
    
    public Worker(String name) {
        this.workerName = name;
    }
    
    public void start() throws Exception {
        ConnectionFactory factory = new ConnectionFactory();
        factory.setHost("localhost");
        
        Connection connection = factory.newConnection();
        Channel channel = connection.createChannel();
        
        // 声明队列（与生产者一致）
        boolean durable = true;
        channel.queueDeclare(QUEUE_NAME, durable, false, false, null);
        
        System.out.println(workerName + " 等待任务...");
        
        // 设置公平分发 - 每次只接收一条消息
        int prefetchCount = 1;
        channel.basicQos(prefetchCount);
        
        // 创建消费者
        DeliverCallback deliverCallback = (consumerTag, delivery) -> {
            String message = new String(delivery.getBody(), StandardCharsets.UTF_8);
            
            System.out.println(workerName + " 接收到: '" + message + "'");
            
            try {
                // 模拟任务处理时间（从消息中提取秒数）
                int seconds = extractProcessingTime(message);
                doWork(seconds);
                
                System.out.println(workerName + " 完成任务: " + message);
                
                // 手动确认消息
                channel.basicAck(delivery.getEnvelope().getDeliveryTag(), false);
                
            } catch (Exception e) {
                System.err.println(workerName + " 处理失败: " + e.getMessage());
                // 拒绝消息，不重新入队
                channel.basicNack(delivery.getEnvelope().getDeliveryTag(), false, false);
            }
        };
        
        // 开始消费，手动确认模式
        boolean autoAck = false;
        channel.basicConsume(QUEUE_NAME, autoAck, deliverCallback, consumerTag -> {});
    }
    
    private int extractProcessingTime(String message) {
        // 从消息中提取处理时间
        // 例如："Task 3: 处理任务需要 3 秒" → 返回3
        String[] parts = message.split("需要 ");
        if (parts.length > 1) {
            String timeStr = parts[1].split(" ")[0];
            return Integer.parseInt(timeStr);
        }
        return 1; // 默认1秒
    }
    
    private void doWork(int seconds) throws InterruptedException {
        // 模拟耗时任务
        Thread.sleep(seconds * 1000L);
    }
    
    public static void main(String[] args) throws Exception {
        Worker worker = new Worker("Worker-" + args[0]);
        worker.start();
        
        // 保持运行
        Thread.sleep(Long.MAX_VALUE);
    }
}
```

## 三、 关键配置：公平分发（Fair Dispatch）

### 1. 问题：不公平分发

默认的循环分发可能导致处理慢的消费者消息堆积，处理快的消费者闲置。

### 2. 解决方案：basicQos

```
// 设置每个消费者一次只处理一条消息
int prefetchCount = 1;
channel.basicQos(prefetchCount);
```

**配置对比**：

```
// ❌ 默认情况（可能不公平）
// 消费者A: 处理耗时任务（消息1、3、5...）
// 消费者B: 处理快速任务（消息2、4、6...）

// ✅ 使用 basicQos
channel.basicQos(1);  // 每次只分发一条消息
// 消费者A: 处理消息1 → 完成后获取消息3
// 消费者B: 处理消息2 → 完成后获取消息4
```

## 四、 Spring Boot 实现工作队列

### 1. 配置类

```
@Configuration
public class WorkQueueConfig {
    
    public static final String WORK_QUEUE = "work.queue";
    
    @Bean
    public Queue workQueue() {
        return QueueBuilder.durable(WORK_QUEUE)
                .withArgument("x-max-priority", 10) // 支持优先级
                .build();
    }
    
    @Bean
    public DirectExchange workExchange() {
        return new DirectExchange("work.exchange");
    }
    
    @Bean
    public Binding workBinding() {
        return BindingBuilder.bind(workQueue())
                .to(workExchange())
                .with("work.routing.key");
    }
    
    // 配置监听器容器工厂
    @Bean
    public SimpleRabbitListenerContainerFactory workQueueFactory(
            ConnectionFactory connectionFactory) {
        SimpleRabbitListenerContainerFactory factory = new SimpleRabbitListenerContainerFactory();
        factory.setConnectionFactory(connectionFactory);
        factory.setConcurrentConsumers(3);    // 最小消费者数
        factory.setMaxConcurrentConsumers(10); // 最大消费者数
        factory.setPrefetchCount(1);          // 公平分发
        factory.setAcknowledgeMode(AcknowledgeMode.MANUAL); // 手动确认
        return factory;
    }
}
```

### 2. 生产者

```
@Component
public class WorkQueueProducer {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    @Autowired
    private AmqpAdmin amqpAdmin;
    
    public void sendTask(int taskId, int processingTime) {
        String message = String.format("任务#%d 需要处理 %d 秒", taskId, processingTime);
        
        MessageProperties props = MessagePropertiesBuilder.newInstance()
                .setDeliveryMode(MessageDeliveryMode.PERSISTENT)
                .setPriority(Math.min(processingTime, 10)) // 设置优先级
                .build();
        
        Message msg = MessageBuilder.withBody(message.getBytes())
                .andProperties(props)
                .build();
        
        rabbitTemplate.send("work.exchange", "work.routing.key", msg);
        
        System.out.println("发送任务: " + message);
    }
    
    // 批量发送任务
    public void sendBatchTasks() {
        for (int i = 1; i <= 20; i++) {
            int processingTime = (i % 5) + 1; // 1-5秒随机
            sendTask(i, processingTime);
            
            try {
                Thread.sleep(200);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }
}
```

### 3. 消费者

```
@Component
public class WorkQueueConsumer {
    
    private static final AtomicInteger workerCounter = new AtomicInteger(1);
    private final String workerName;
    
    public WorkQueueConsumer() {
        this.workerName = "Worker-" + workerCounter.getAndIncrement();
    }
    
    @RabbitListener(
        queues = WorkQueueConfig.WORK_QUEUE,
        containerFactory = "workQueueFactory"
    )
    public void handleTask(String message, Channel channel, 
                          @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
        
        System.out.println(workerName + " 开始处理: " + message);
        
        try {
            // 提取处理时间
            int seconds = extractProcessingTime(message);
            
            // 模拟任务处理
            processTask(seconds);
            
            // 确认消息
            channel.basicAck(deliveryTag, false);
            
            System.out.println(workerName + " 完成: " + message);
            
        } catch (Exception e) {
            System.err.println(workerName + " 处理失败: " + e.getMessage());
            
            try {
                // 拒绝消息，可选择是否重新入队
                boolean requeue = shouldRequeue(e);
                channel.basicNack(deliveryTag, false, requeue);
            } catch (IOException ioException) {
                ioException.printStackTrace();
            }
        }
    }
    
    private int extractProcessingTime(String message) {
        Pattern pattern = Pattern.compile("需要处理 (\\d+) 秒");
        Matcher matcher = pattern.matcher(message);
        if (matcher.find()) {
            return Integer.parseInt(matcher.group(1));
        }
        return 1;
    }
    
    private void processTask(int seconds) throws InterruptedException {
        // 模拟耗时操作
        Thread.sleep(seconds * 1000L);
        
        // 模拟10%的失败率
        if (Math.random() < 0.1) {
            throw new RuntimeException("任务处理失败");
        }
    }
    
    private boolean shouldRequeue(Exception e) {
        // 根据异常类型决定是否重新入队
        return !(e instanceof IllegalArgumentException);
    }
}
```

## 五、 高级特性

### 1. 消息优先级

```
// 声明支持优先级的队列
Map<String, Object> args = new HashMap<>();
args.put("x-max-priority", 10); // 最大优先级10
channel.queueDeclare("priority.queue", true, false, false, args);

// 发送高优先级消息
AMQP.BasicProperties highPriority = new AMQP.BasicProperties.Builder()
    .priority(5)  // 设置优先级
    .build();
channel.basicPublish("", "priority.queue", highPriority, message.getBytes());
```

### 2. 消息确认与可靠性

```
// 自动确认（不推荐用于工作队列）
boolean autoAck = true; // 消息接收后立即确认

// 手动确认（推荐）
boolean autoAck = false;
channel.basicConsume(QUEUE_NAME, autoAck, consumer);

// 在消费者中确认
channel.basicAck(deliveryTag, false); // 成功
channel.basicNack(deliveryTag, false, true); // 失败，重新入队
channel.basicNack(deliveryTag, false, false); // 失败，丢弃
```

### 3. 消息持久化

```
// 队列持久化
boolean durable = true;
channel.queueDeclare(QUEUE_NAME, durable, false, false, null);

// 消息持久化
AMQP.BasicProperties props = new AMQP.BasicProperties.Builder()
    .deliveryMode(2) // 2=持久化
    .build();
channel.basicPublish("", QUEUE_NAME, props, message.getBytes());
```

## 七、 最佳实践

1. **始终启用消息持久化**：防止RabbitMQ重启时消息丢失
    
2. **使用公平分发**：`basicQos(1)`确保负载均衡
    
3. **手动确认消息**：确保消息被正确处理
    
4. **处理消费者失败**：实现重试和死信队列
    
5. **监控队列长度**：防止消息无限堆积
    
6. **设置合理的TTL**：避免旧消息堆积
    
7. **实现消费者幂等性**：防止重复消费
    
8. **优雅关闭**：消费者关闭前完成正在处理的任务
    

## 八、 常见问题

### Q1: 消息丢失怎么办？

- 启用消息持久化
    
- 使用发布者确认
    
- 手动确认消息
    

### Q2: 消费者处理太慢？

- 增加消费者数量
    
- 优化消费者处理逻辑
    
- 使用消息优先级
    

### Q3: 消息重复消费？

- 实现消费者幂等性
    
- 使用消息ID去重
    
- 数据库唯一约束
    

### Q4: 消费者宕机如何处理？

- 设置合理的超时时间
    
- 实现消费者健康检查
    
- 使用连接心跳
    

工作队列模式是 RabbitMQ 最常用的场景之一，合理配置可以实现高效、可靠的任务处理系统。
