在 Java 中配置 RabbitMQ 主要分为两种方式：**原生 AMQP 客户端**​ 和 **Spring Boot 集成**。以下是详细的配置示例。

## 一、 原生 AMQP 客户端配置

### 1. 添加 Maven 依赖

```
<dependency>
    <groupId>com.rabbitmq</groupId>
    <artifactId>amqp-client</artifactId>
    <version>5.20.0</version>
</dependency>
```

### 2. 基础连接配置

```
import com.rabbitmq.client.Connection;
import com.rabbitmq.client.ConnectionFactory;

public class RabbitMQConfig {
    
    public static Connection getConnection() throws Exception {
        ConnectionFactory factory = new ConnectionFactory();
        
        // 基础配置
        factory.setHost("localhost");      // RabbitMQ服务器地址
        factory.setPort(5672);              // 端口，默认5672
        factory.setVirtualHost("/");        // 虚拟主机，默认/
        factory.setUsername("guest");       // 用户名，默认guest
        factory.setPassword("guest");       // 密码，默认guest
        
        // 高级配置
        factory.setConnectionTimeout(30000); // 连接超时时间(ms)
        factory.setAutomaticRecoveryEnabled(true); // 自动重连
        factory.setNetworkRecoveryInterval(5000);  // 重连间隔(ms)
        
        // 创建连接
        return factory.newConnection();
    }
}
```

### 3. 生产者示例

```
import com.rabbitmq.client.Channel;
import com.rabbitmq.client.Connection;

public class ProducerExample {
    
    private static final String QUEUE_NAME = "test.queue";
    private static final String EXCHANGE_NAME = "test.exchange";
    private static final String ROUTING_KEY = "test.routing.key";
    
    public void sendMessage(String message) throws Exception {
        try (Connection connection = RabbitMQConfig.getConnection();
             Channel channel = connection.createChannel()) {
            
            // 1. 声明交换机（可选，如果已存在可省略）
            // 参数：交换机名，交换机类型，是否持久化，是否自动删除，其他参数
            channel.exchangeDeclare(EXCHANGE_NAME, "direct", true, false, null);
            
            // 2. 声明队列
            // 参数：队列名，是否持久化，是否排他，是否自动删除，其他参数
            Map<String, Object> arguments = new HashMap<>();
            arguments.put("x-message-ttl", 60000); // 消息TTL：60秒
            channel.queueDeclare(QUEUE_NAME, true, false, false, arguments);
            
            // 3. 绑定队列到交换机
            channel.queueBind(QUEUE_NAME, EXCHANGE_NAME, ROUTING_KEY);
            
            // 4. 发布消息
            // 参数：交换机，路由键，属性，消息体
            AMQP.BasicProperties props = new AMQP.BasicProperties.Builder()
                .deliveryMode(2)      // 2=持久化消息
                .contentType("text/plain")
                .build();
            
            channel.basicPublish(EXCHANGE_NAME, ROUTING_KEY, props, 
                                  message.getBytes(StandardCharsets.UTF_8));
            
            System.out.println("消息发送成功: " + message);
        }
    }
}
```

### 4. 消费者示例

```
public class ConsumerExample {
    
    private static final String QUEUE_NAME = "test.queue";
    
    public void consumeMessage() throws Exception {
        Connection connection = RabbitMQConfig.getConnection();
        Channel channel = connection.createChannel();
        
        // 设置QoS（服务质量），每次只接收一条消息
        channel.basicQos(1);
        
        // 声明队列（确保队列存在）
        channel.queueDeclare(QUEUE_NAME, true, false, false, null);
        
        // 创建消费者
        DefaultConsumer consumer = new DefaultConsumer(channel) {
            @Override
            public void handleDelivery(String consumerTag, Envelope envelope,
                                       AMQP.BasicProperties properties, byte[] body) {
                try {
                    String message = new String(body, StandardCharsets.UTF_8);
                    System.out.println("接收到消息: " + message);
                    
                    // 手动确认消息
                    // 参数：消息标签，是否批量确认
                    channel.basicAck(envelope.getDeliveryTag(), false);
                    
                } catch (Exception e) {
                    // 处理失败，拒绝消息
                    // 参数：消息标签，是否重新入队
                    channel.basicNack(envelope.getDeliveryTag(), false, true);
                }
            }
        };
        
        // 开始消费
        // 参数：队列名，是否自动确认，消费者
        channel.basicConsume(QUEUE_NAME, false, consumer);
    }
}
```

## 二、 Spring Boot 集成配置

### 1. 添加 Maven 依赖

```
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-amqp</artifactId>
</dependency>
```

### 2. 配置文件（application.yml）

```
spring:
  rabbitmq:
    # 基础连接配置
    host: localhost
    port: 5672
    username: guest
    password: guest
    virtual-host: /
    
    # 连接池配置
    connection-timeout: 30000
    cache:
      channel:
        size: 25  # 通道缓存大小
      connection:
        mode: channel
        size: 5   # 连接缓存大小
    
    # 监听器配置
    listener:
      simple:
        acknowledge-mode: manual  # 手动确认
        concurrency: 3           # 最小消费者数
        max-concurrency: 10      # 最大消费者数
        prefetch: 1              # 每次处理一条
        retry:
          enabled: true          # 开启重试
          max-attempts: 3        # 最大重试次数
          initial-interval: 2000ms # 重试间隔
    
    # 高级配置
    publisher-confirm-type: correlated  # 发布确认
    publisher-returns: true             # 开启返回模式
```

### 3. RabbitMQ 配置类

```
import org.springframework.amqp.core.*;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class RabbitMQConfig {
    
    // 1. 定义队列
    @Bean
    public Queue testQueue() {
        Map<String, Object> args = new HashMap<>();
        args.put("x-message-ttl", 60000);        // 消息TTL
        args.put("x-max-length", 10000);        // 队列最大长度
        args.put("x-dead-letter-exchange", "dlx.exchange"); // 死信交换机
        return QueueBuilder.durable("test.queue")  // 持久化队列
                .withArguments(args)
                .build();
    }
    
    // 2. 定义交换机
    @Bean
    public DirectExchange testExchange() {
        return ExchangeBuilder.directExchange("test.exchange")
                .durable(true)    // 持久化
                .build();
    }
    
    // 3. 定义绑定
    @Bean
    public Binding testBinding() {
        return BindingBuilder.bind(testQueue())
                .to(testExchange())
                .with("test.routing.key");
    }
    
    // 4. 死信交换机配置
    @Bean
    public DirectExchange dlxExchange() {
        return new DirectExchange("dlx.exchange", true, false);
    }
    
    @Bean
    public Queue dlxQueue() {
        return new Queue("dlx.queue", true);
    }
    
    @Bean
    public Binding dlxBinding() {
        return BindingBuilder.bind(dlxQueue())
                .to(dlxExchange())
                .with("dlx.routing.key");
    }
}
```

### 4. 生产者（Spring Boot）

```
import org.springframework.amqp.rabbit.core.RabbitTemplate;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

@Component
public class RabbitMQProducer {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    public void sendMessage(String message) {
        // 简单发送
        rabbitTemplate.convertAndSend("test.exchange", "test.routing.key", message);
        
        // 高级发送（带属性）
        MessageProperties props = MessagePropertiesBuilder.newInstance()
                .setContentType(MessageProperties.CONTENT_TYPE_TEXT_PLAIN)
                .setDeliveryMode(MessageDeliveryMode.PERSISTENT)
                .setHeader("custom-header", "value")
                .build();
        
        Message msg = MessageBuilder.withBody(message.getBytes())
                .andProperties(props)
                .build();
        
        rabbitTemplate.send("test.exchange", "test.routing.key", msg);
    }
}
```

## 三、 重要配置项说明

### 1. 消息确认模式

```
spring.rabbitmq.listener.simple.acknowledge-mode:
  - none:   自动确认（不推荐，可能丢失消息）
  - auto:   自动确认，基于业务是否抛异常
  - manual: 手动确认（推荐，需调用basicAck）
```

### 2. 事务配置

```
// 开启事务
rabbitTemplate.setChannelTransacted(true);

// 事务发送
rabbitTemplate.execute(channel -> {
    // 多条消息发送
    rabbitTemplate.convertAndSend("exchange1", "key1", "message1");
    rabbitTemplate.convertAndSend("exchange2", "key2", "message2");
    return null;
});
```

### 3. 消息可靠性配置

```
@Configuration
public class RabbitMQReliabilityConfig {
    
    // 1. 确认回调
    @Bean
    public RabbitTemplate.ConfirmCallback confirmCallback() {
        return (correlationData, ack, cause) -> {
            if (ack) {
                System.out.println("消息发送到交换机成功");
            } else {
                System.out.println("消息发送失败: " + cause);
            }
        };
    }
    
    // 2. 返回回调（路由失败回调）
    @Bean
    public RabbitTemplate.ReturnsCallback returnsCallback() {
        return returned -> {
            System.out.println("消息路由到队列失败: " + returned.getReplyText());
        };
    }
    
    @Bean
    public RabbitTemplate rabbitTemplate(ConnectionFactory connectionFactory) {
        RabbitTemplate template = new RabbitTemplate(connectionFactory);
        template.setMandatory(true); // 开启返回模式
        template.setConfirmCallback(confirmCallback());
        template.setReturnsCallback(returnsCallback());
        return template;
    }
}
```

## 四、 最佳实践建议

1. **连接管理**：使用连接池，避免频繁创建连接
    
2. **消息持久化**：队列和消息都设置为持久化
    
3. **手动确认**：使用手动ACK，确保消息不丢失
    
4. **设置TTL**：为消息和队列设置合理的过期时间
    
5. **死信队列**：配置死信交换机处理失败消息
    
6. **监控告警**：集成监控，关注队列堆积情况
    
7. **幂等性**：消费者端实现消息处理的幂等性
    

以上配置涵盖了 Java 中使用 RabbitMQ 的主要方式，实际使用中可根据具体业务需求调整配置参数。
