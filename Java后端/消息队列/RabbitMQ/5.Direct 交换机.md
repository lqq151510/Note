# RabbitMQ Direct 交换机

Direct（直连）交换机是 RabbitMQ 中最基本、最常用的交换机类型，它根据 **Routing Key 的精确匹配**​ 将消息路由到队列，实现**点对点**和**路由选择**功能。

## 一、 核心特性

### 1. 工作模式

```
[Direct Exchange]
              │
      ┌───────┼───────┐
      ▼       ▼       ▼
  [Queue1] [Queue2] [Queue3]
   (error)  (info)  (warning)
      │       │       │
RoutingKey: error   info   warning
```

- **精确匹配**：消息的 Routing Key 必须与 Binding Key **完全一致**
    
- **多对一/多**：一个队列可以绑定多个 Routing Key，多个队列也可以绑定相同的 Routing Key
    
- **单播/多播**：可以实现精确的单播，也可以实现选择性多播
    

### 2. 路由规则

```
// 绑定关系
channel.queueBind("queue1", "direct_exchange", "error");
channel.queueBind("queue2", "direct_exchange", "info");
channel.queueBind("queue3", "direct_exchange", "warning");
channel.queueBind("queue4", "direct_exchange", "error"); // 多个队列可绑定相同key

// 发送消息
channel.basicPublish("direct_exchange", "error", ...);   // 发送到 queue1 和 queue4
channel.basicPublish("direct_exchange", "info", ...);    // 发送到 queue2
channel.basicPublish("direct_exchange", "debug", ...);   // 无队列接收，消息丢弃
```

## 二、 基础实现示例

### 1. 生产者

```
import com.rabbitmq.client.Channel;
import com.rabbitmq.client.Connection;
import com.rabbitmq.client.ConnectionFactory;
import java.nio.charset.StandardCharsets;

public class DirectProducer {
    
    private static final String EXCHANGE_NAME = "direct_exchange";
    // 定义 Routing Key
    private static final String[] ROUTING_KEYS = {"error", "warning", "info", "debug"};
    
    public static void main(String[] args) throws Exception {
        ConnectionFactory factory = new ConnectionFactory();
        factory.setHost("localhost");
        
        try (Connection connection = factory.newConnection();
             Channel channel = connection.createChannel()) {
            
            // 1. 声明 Direct 交换机
            channel.exchangeDeclare(EXCHANGE_NAME, "direct", true);
            
            // 2. 发送不同级别的日志消息
            for (int i = 1; i <= 10; i++) {
                String routingKey = getRandomRoutingKey();
                String message = String.format("日志#%d [%s]: 这是%s级别的日志消息", 
                    i, routingKey.toUpperCase(), routingKey);
                
                // 发送消息，指定 Routing Key
                channel.basicPublish(EXCHANGE_NAME, routingKey, null, 
                    message.getBytes(StandardCharsets.UTF_8));
                
                System.out.printf(" [x] 发送到 '%s': %s%n", routingKey, message);
                Thread.sleep(500);
            }
            
            // 3. 发送特定消息
            sendSpecificMessage(channel, "error", "系统发生严重错误！");
            sendSpecificMessage(channel, "info", "系统启动完成");
            sendSpecificMessage(channel, "warning", "磁盘空间不足警告");
        }
    }
    
    private static String getRandomRoutingKey() {
        return ROUTING_KEYS[(int) (Math.random() * ROUTING_KEYS.length)];
    }
    
    private static void sendSpecificMessage(Channel channel, String routingKey, String message) 
            throws Exception {
        channel.basicPublish(EXCHANGE_NAME, routingKey, null, 
            message.getBytes(StandardCharsets.UTF_8));
        System.out.printf(" [x] 特定消息到 '%s': %s%n", routingKey, message);
    }
}
```

### 2. 消费者

```
import com.rabbitmq.client.*;
import java.nio.charset.StandardCharsets;

public class DirectConsumer {
    
    private static final String EXCHANGE_NAME = "direct_exchange";
    private final String consumerName;
    private final String[] routingKeys; // 这个消费者关心的 Routing Key
    
    public DirectConsumer(String name, String... routingKeys) {
        this.consumerName = name;
        this.routingKeys = routingKeys;
    }
    
    public void start() throws Exception {
        ConnectionFactory factory = new ConnectionFactory();
        factory.setHost("localhost");
        
        Connection connection = factory.newConnection();
        Channel channel = connection.createChannel();
        
        // 1. 声明 Direct 交换机
        channel.exchangeDeclare(EXCHANGE_NAME, "direct", true);
        
        // 2. 创建队列（随机名称，非持久化，排他，自动删除）
        String queueName = channel.queueDeclare().getQueue();
        System.out.println(consumerName + " 队列名: " + queueName);
        
        // 3. 绑定队列到交换机，指定关心的 Routing Keys
        for (String routingKey : routingKeys) {
            channel.queueBind(queueName, EXCHANGE_NAME, routingKey);
            System.out.println(consumerName + " 绑定到: " + routingKey);
        }
        
        System.out.println(consumerName + " 等待消息...");
        
        // 4. 创建消费者回调
        DeliverCallback deliverCallback = (consumerTag, delivery) -> {
            String message = new String(delivery.getBody(), StandardCharsets.UTF_8);
            String routingKey = delivery.getEnvelope().getRoutingKey();
            
            System.out.printf("%s 收到 [%s]: %s%n", consumerName, routingKey, message);
            
            // 模拟不同处理逻辑
            processMessage(routingKey, message);
        };
        
        // 5. 开始消费
        channel.basicConsume(queueName, true, deliverCallback, consumerTag -> {});
    }
    
    private void processMessage(String routingKey, String message) throws InterruptedException {
        // 根据不同的 Routing Key 进行不同处理
        switch (routingKey) {
            case "error":
                System.out.println(consumerName + " 处理错误日志...");
                Thread.sleep(2000); // 错误处理较慢
                break;
            case "warning":
                System.out.println(consumerName + " 处理警告日志...");
                Thread.sleep(1000);
                break;
            case "info":
            case "debug":
                System.out.println(consumerName + " 处理信息日志...");
                Thread.sleep(500);
                break;
        }
    }
    
    public static void main(String[] args) throws Exception {
        // 启动多个消费者，每个关注不同的 Routing Key
        Thread errorConsumer = new Thread(() -> {
            try {
                new DirectConsumer("错误处理器", "error").start();
            } catch (Exception e) {
                e.printStackTrace();
            }
        });
        
        Thread warningConsumer = new Thread(() -> {
            try {
                new DirectConsumer("警告处理器", "warning").start();
            } catch (Exception e) {
                e.printStackTrace();
            }
        });
        
        Thread infoConsumer = new Thread(() -> {
            try {
                new DirectConsumer("信息处理器", "info", "debug").start();
            } catch (Exception e) {
                e.printStackTrace();
            }
        });
        
        Thread allConsumer = new Thread(() -> {
            try {
                new DirectConsumer("全量处理器", "error", "warning", "info", "debug").start();
            } catch (Exception e) {
                e.printStackTrace();
            }
        });
        
        errorConsumer.start();
        warningConsumer.start();
        infoConsumer.start();
        allConsumer.start();
        
        // 保持运行
        Thread.sleep(30000);
    }
}
```

### 2. 生产者

```
@Component
public class DirectProducer {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    @Autowired
    private DirectExchange directExchange;
    
    // 发送日志消息
    public void sendLogMessage(String level, String message) {
        String routingKey = level.toLowerCase(); // error, warning, info, debug
        
        // 设置消息属性
        MessageProperties props = MessagePropertiesBuilder.newInstance()
                .setContentType(MessageProperties.CONTENT_TYPE_TEXT_PLAIN)
                .setDeliveryMode(MessageDeliveryMode.PERSISTENT)
                .setPriority(getPriority(level))
                .setHeader("timestamp", System.currentTimeMillis())
                .setHeader("level", level)
                .build();
        
        String formattedMessage = String.format("[%s] %s - %s", 
                LocalDateTime.now().format(DateTimeFormatter.ISO_LOCAL_TIME),
                level.toUpperCase(), message);
        
        Message msg = MessageBuilder.withBody(formattedMessage.getBytes())
                .andProperties(props)
                .build();
        
        rabbitTemplate.send(directExchange.getName(), routingKey, msg);
        
        System.out.println("发送日志 [" + level + "]: " + message);
    }
    
    // 发送订单状态消息
    public void sendOrderStatus(String orderId, String status) {
        String routingKey = "order." + status; // order.created, order.paid, order.shipped
        
        OrderEvent event = new OrderEvent();
        event.setOrderId(orderId);
        event.setStatus(status);
        event.setTimestamp(new Date());
        
        rabbitTemplate.convertAndSend(directExchange.getName(), routingKey, event);
        System.out.println("发送订单状态: " + orderId + " -> " + status);
    }
    
    // 发送用户消息
    public void sendUserMessage(String userId, String type, String content) {
        String routingKey = "user." + type; // user.notification, user.message, user.alert
        
        Map<String, Object> message = new HashMap<>();
        message.put("userId", userId);
        message.put("type", type);
        message.put("content", content);
        message.put("timestamp", System.currentTimeMillis());
        
        rabbitTemplate.convertAndSend(directExchange.getName(), routingKey, message);
        System.out.println("发送用户消息 [" + type + "] 给用户: " + userId);
    }
    
    // 批量发送
    public void sendBatchMessages() {
        // 发送各种级别的日志
        sendLogMessage("error", "数据库连接失败");
        sendLogMessage("warning", "内存使用率超过80%");
        sendLogMessage("info", "用户登录成功");
        sendLogMessage("debug", "请求处理开始");
        
        // 发送订单状态
        sendOrderStatus("ORD001", "created");
        sendOrderStatus("ORD001", "paid");
        sendOrderStatus("ORD001", "shipped");
        
        // 发送用户消息
        sendUserMessage("USER001", "notification", "您有新的订单");
        sendUserMessage("USER001", "message", "系统维护通知");
    }
    
    private int getPriority(String level) {
        switch (level.toLowerCase()) {
            case "error": return 5;
            case "warning": return 3;
            case "info": return 1;
            case "debug": return 0;
            default: return 0;
        }
    }
    
    @Data
    @NoArgsConstructor
    @AllArgsConstructor
    public static class OrderEvent {
        private String orderId;
        private String status;
        private Date timestamp;
    }
}
```

```
@Component
public class ErrorLogConsumer {
    
    @RabbitListener(queues = "direct.queue.error")
    public void handleErrorLog(String message,
                              @Header(AmqpHeaders.RECEIVED_ROUTING_KEY) String routingKey,
                              Channel channel,
                              @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
        
        System.out.println("[错误日志处理器] 收到消息: " + message);
        System.out.println("Routing Key: " + routingKey);
        
        try {
            // 错误日志的特殊处理逻辑
            sendErrorAlert(message);
            saveToErrorDatabase(message);
            
            // 手动确认
            channel.basicAck(deliveryTag, false);
            
        } catch (Exception e) {
            System.err.println("处理错误日志失败: " + e.getMessage());
            try {
                // 消息处理失败，重新入队
                channel.basicNack(deliveryTag, false, true);
            } catch (IOException ioException) {
                ioException.printStackTrace();
            }
        }
    }
    
    private void sendErrorAlert(String message) {
        // 发送错误告警
        System.out.println("发送错误告警: " + message);
    }
    
    private void saveToErrorDatabase(String message) {
        // 保存到错误日志数据库
        System.out.println("保存到错误日志数据库: " + message);
    }
}
```

## 四、 动态绑定与路由管理

### 1. 动态绑定服务

```
@Service
public class DynamicBindingService {
    
    @Autowired
    private AmqpAdmin amqpAdmin;
    
    @Autowired
    private DirectExchange directExchange;
    
    /**
     * 动态创建队列并绑定
     */
    public String createAndBindQueue(String queueName, String routingKey) {
        // 创建队列
        Queue queue = QueueBuilder.durable(queueName)
                .withArgument("x-queue-mode", "lazy")
                .build();
        
        amqpAdmin.declareQueue(queue);
        
        // 创建绑定
        Binding binding = new Binding(
            queueName,
            Binding.DestinationType.QUEUE,
            directExchange.getName(),
            routingKey,
            null
        );
        
        amqpAdmin.declareBinding(binding);
        
        System.out.println("队列 " + queueName + " 已绑定到 " + routingKey);
        return queueName;
    }
    
    /**
     * 为现有队列添加新的 Routing Key 绑定
     */
    public void addRoutingKey(String queueName, String newRoutingKey) {
        Binding binding = new Binding(
            queueName,
            Binding.DestinationType.QUEUE,
            directExchange.getName(),
            newRoutingKey,
            null
        );
        
        amqpAdmin.declareBinding(binding);
        System.out.println("队列 " + queueName + " 添加绑定: " + newRoutingKey);
    }
    
    /**
     * 移除队列的某个 Routing Key 绑定
     */
    public void removeRoutingKey(String queueName, String routingKey) {
        Binding binding = new Binding(
            queueName,
            Binding.DestinationType.QUEUE,
            directExchange.getName(),
            routingKey,
            null
        );
        
        amqpAdmin.removeBinding(binding);
        System.out.println("队列 " + queueName + " 移除绑定: " + routingKey);
    }
    
    /**
     * 获取交换机的所有绑定
     */
    public List<Map<String, Object>> getExchangeBindings() {
        return rabbitTemplate.execute(channel -> {
            List<Map<String, Object>> bindings = new ArrayList<>();
            
            // 获取绑定列表
            AMQP.Queue.BindOk[] bindingsArray = channel.queueBindings(directExchange.getName());
            
            for (AMQP.Queue.BindOk binding : bindingsArray) {
                Map<String, Object> bindingInfo = new HashMap<>();
                bindingInfo.put("queue", binding.getQueue());
                bindingInfo.put("routingKey", binding.getRoutingKey());
                bindingInfo.put("exchange", binding.getExchange());
                bindings.add(bindingInfo);
            }
            
            return bindings;
        });
    }
}
```

### 2. 路由表管理

```
@Service
public class RoutingService {
    
    private final Map<String, List<String>> routingTable = new ConcurrentHashMap<>();
    
    @Autowired
    private DynamicBindingService dynamicBindingService;
    
    /**
     * 注册路由规则
     */
    public void registerRoute(String serviceName, String routingKey, String queueName) {
        // 创建队列并绑定
        String actualQueueName = dynamicBindingService.createAndBindQueue(queueName, routingKey);
        
        // 更新路由表
        routingTable.computeIfAbsent(serviceName, k -> new ArrayList<>())
                   .add(routingKey + "->" + actualQueueName);
        
        System.out.println("注册路由: " + serviceName + " -> " + routingKey + " -> " + actualQueueName);
    }
    
    /**
     * 发送消息到特定服务
     */
    public void sendToService(String serviceName, String message) {
        List<String> routes = routingTable.get(serviceName);
        if (routes != null) {
            for (String route : routes) {
                String[] parts = route.split("->");
                if (parts.length == 2) {
                    String routingKey = parts[0];
                    String queueName = parts[1];
                    
                    // 实际发送消息
                    sendMessage(routingKey, message, serviceName);
                }
            }
        }
    }
    
    /**
     * 根据条件动态路由
     */
    public void dynamicRoute(String messageType, String messageContent) {
        String routingKey = determineRoutingKey(messageType, messageContent);
        
        if (routingKey != null) {
            sendMessage(routingKey, messageContent, "dynamic");
        }
    }
    
    private String determineRoutingKey(String messageType, String content) {
        // 根据消息类型和内容决定路由键
        if (messageType.equals("ERROR")) {
            if (content.contains("critical")) {
                return "error.critical";
            } else {
                return "error.normal";
            }
        } else if (messageType.equals("ORDER")) {
            if (content.contains("created")) {
                return "order.created";
            } else if (content.contains("paid")) {
                return "order.paid";
            }
        }
        return "default";
    }
    
    private void sendMessage(String routingKey, String message, String source) {
        // 实际发送逻辑
        System.out.println("路由消息 [" + source + "] -> " + routingKey + ": " + message);
    }
}
```

```
                .to(mainExchange())
                .with("delayed");
```

## 八、 最佳实践

1. **命名规范**：
    
    - 使用有意义的 Routing Key，如 `order.created`、`user.registered`
        
    - 使用点号分隔层次，如 `system.log.error`、`notification.email`
        
    
2. **错误处理**：
    
    - 为重要消息配置死信队列
        
    - 实现消息重试机制
        
    - 监控未路由的消息
        
    
3. **性能优化**：
    
    - 避免一个队列绑定过多 Routing Key
        
    - 使用连接池和信道复用
        
    - 合理设置 prefetch count
        
    
4. **可靠性**：
    
    - 重要队列设置为持久化
        
    - 使用发布者确认机制
        
    - 实现消费者幂等性
        
    
5. **监控告警**：
    
    - 监控队列长度和消费者数量
        
    - 设置消息堆积告警
        
    - 监控消息处理延迟
        
    

## 九、 常见问题

### Q1: Direct 和 Fanout 的区别？

- **Direct**：精确匹配 Routing Key，实现选择性路由
    
- **Fanout**：忽略 Routing Key，广播到所有队列
    

### Q2: 消息没有消费者时会发生什么？

- 消息会一直留在队列中等待消费者
    
- 可以设置 TTL 让消息自动过期
    
- 建议监控无人消费的队列
    

### Q3: 如何实现消息优先级？

- 队列声明时设置 `x-max-priority`参数
    
- 发送消息时设置 `priority`属性
    
- 注意：需要消费者支持才能按优先级处理
    

### Q4: 多个队列绑定相同 Routing Key？

- 可以，消息会复制到所有绑定的队列
    
- 适用于多消费者处理相同消息的场景
    
- 注意消息重复处理问题
    

Direct 交换机是 RabbitMQ 中最灵活、最常用的交换机类型，适合需要精确路由控制的场景。通过合理的 Routing Key 设计，可以构建出清晰、可维护的消息路由系统。
