# RabbitMQ 业务幂等性

**业务幂等性**是消息队列系统中的**核心安全机制**，确保同一消息被多次处理时，业务结果保持一致，防止重复消费导致的数据不一致。

## 一、 幂等性基础原理

### 1. 幂等性定义

```
/**
 * 幂等性接口定义
 * 对于同一个操作，执行一次和执行多次的结果是一样的
 */
public interface IdempotentService {
    
    /**
     * 幂等方法示例
     * 相同输入总是产生相同输出
     */
    String processOrder(String orderId, String action);
    
    /**
     * 非幂等方法示例
     * 相同输入可能产生不同输出
     */
    void updateBalance(String accountId, BigDecimal amount);
}
```

### 2. 幂等性场景分析

```
@Component
public class IdempotentScenario {
    
    /**
     * 场景1: 天然幂等操作
     * SELECT查询、DELETE、UPDATE带条件
     */
    public boolean isNaturalIdempotent(String operation) {
        return operation.matches("SELECT|DELETE.*WHERE.*|UPDATE.*WHERE.*");
    }
    
    /**
     * 场景2: 非幂等操作
     * INSERT、UPDATE不带条件、复杂计算
     */
    public boolean isNonIdempotent(String operation) {
        return operation.matches("INSERT|UPDATE|.*[+\\-*/].*");
    }
    
    /**
     * 幂等操作示例
     */
    public class IdempotentOperations {
        // 设置操作（天然幂等）
        public void setUserStatus(String userId, String status) {
            // UPDATE users SET status = ? WHERE user_id = ?

        // 查询操作（天然幂等）
        public User getUser(String userId) {
            // SELECT * FROM users WHERE user_id = ?
        }
        
        // 删除操作（天然幂等）
        public void deleteOrder(String orderId) {
            // DELETE FROM orders WHERE order_id = ?
        }

     * 非幂等操作示例
     */
    public class NonIdempotentOperations {
```

```
@Component
public class UniqueMessageIdStrategy {
    
    private final Set<String> processedMessageIds = Collections.newSetFromMap(
        new ConcurrentHashMap<>()
    );
    
    private final Cache<String, Boolean> messageIdCache = Caffeine.newBuilder()
        .maximumSize(100000)  // 最大缓存10万条
        .expireAfterWrite(1, TimeUnit.HOURS)  // 1小时后过期
        .build();
    
    /**
     * 基于内存Set的幂等性校验
     */
    public boolean checkIdempotent(String messageId) {
        // 检查是否已处理
        if (processedMessageIds.contains(messageId)) {
            System.out.println("消息已处理: " + messageId);
            return false;
        }
        
        // 标记为已处理
        processedMessageIds.add(messageId);
        return true;
    }
    
    /**
     * 基于Caffeine缓存的幂等性校验
     */
    public boolean checkIdempotentWithCache(String messageId) {
        // 检查缓存
        Boolean processed = messageIdCache.getIfPresent(messageId);
        if (Boolean.TRUE.equals(processed)) {
            System.out.println("消息已处理（缓存）: " + messageId);
            return false;
        }
        
        // 标记为已处理
        messageIdCache.put(messageId, true);
        return true;
    }
    
    /**
     * 分布式幂等性校验（Redis实现）
     */
    public boolean checkIdempotentDistributed(String messageId, RedisTemplate<String, String> redisTemplate) {
        String key = "msg:idempotent:" + messageId;
        
        // 使用SETNX原子操作
        Boolean success = redisTemplate.opsForValue().setIfAbsent(key, "1", 1, TimeUnit.HOURS);
        
        if (Boolean.FALSE.equals(success)) {
            System.out.println("消息已处理（分布式）: " + messageId);
            return false;
        }
        
        return true;
    }
    
    /**
     * 带业务标识的幂等性校验
     */
    public boolean checkIdempotentWithBiz(String businessType, String businessKey) {
        String compositeKey = businessType + ":" + businessKey;
        return checkIdempotent(compositeKey);
    }
    
    /**
     * 批量幂等性校验
     */
    public List<String> filterProcessedMessages(List<String> messageIds) {
        List<String> unprocessed = new ArrayList<>();
        
        for (String messageId : messageIds) {
            if (!processedMessageIds.contains(messageId)) {
                unprocessed.add(messageId);
                processedMessageIds.add(messageId);
            }
        }
        
        return unprocessed;
    }
    
    /**
     * 清理过期消息ID
     */
    @Scheduled(fixedDelay = 3600000)  // 每小时清理一次
    public void cleanupExpiredMessages() {
        // 可以根据业务需求清理
        if (processedMessageIds.size() > 100000) {
            // 清理策略：可以清理最早的一半
            // 或者使用LRU策略
        }
    }
}
```

```
@Component
public class DatabaseUniqueConstraintStrategy {
    
    @Autowired
    private JdbcTemplate jdbcTemplate;
    
    /**
     * 消息处理记录表
     * CREATE TABLE message_processed (
     *     id BIGINT PRIMARY KEY AUTO_INCREMENT,
     *     message_id VARCHAR(64) NOT NULL UNIQUE,
     *     business_type VARCHAR(32) NOT NULL,
     *     business_key VARCHAR(128) NOT NULL,
     *     processed_time DATETIME NOT NULL,
     *     INDEX idx_business (business_type, business_key)
     * );
     */
    
    /**
     * 插入记录实现幂等性
     */
    public boolean processWithIdempotent(String messageId, String businessType, 
                                        String businessKey, Runnable processor) {
        
        try {
            // 1. 尝试插入处理记录
            insertProcessRecord(messageId, businessType, businessKey);
            
            // 2. 执行业务逻辑
            processor.run();
            
            return true;
            
        } catch (DuplicateKeyException e) {
            // 唯一约束冲突，说明已处理
            System.out.println("消息已处理（数据库约束）: " + messageId);
            return false;
            
        } catch (Exception e) {
            // 其他异常，回滚处理记录
            rollbackProcessRecord(messageId);
            throw e;
        }
    }
    
    /**
     * 先查后插模式
     */
    public boolean processWithCheckFirst(String messageId, String businessType, 
                                        String businessKey, Runnable processor) {
        
        // 1. 先检查是否已处理
        if (isMessageProcessed(messageId, businessType, businessKey)) {
            System.out.println("消息已处理（检查）: " + messageId);
            return false;
        }
        
        try {
            // 2. 插入处理记录
            insertProcessRecord(messageId, businessType, businessKey);
            
            // 3. 执行业务逻辑
            processor.run();
            
            return true;
            
        } catch (DuplicateKeyException e) {
            // 并发情况下的重复处理
            System.out.println("消息并发处理: " + messageId);
            return false;
        }
    }
    
    /**
     * 使用数据库事务
     */
    @Transactional
    public boolean processInTransaction(String messageId, String businessType, 
                                       String businessKey, Runnable processor) {
        
        // 检查是否已处理
        if (isMessageProcessed(messageId, businessType, businessKey)) {
            return false;
        }
        
        // 插入记录
        insertProcessRecord(messageId, businessType, businessKey);
        
        // 执行业务逻辑
        processor.run();
        
        return true;
    }
    
    /**
     * 批量处理幂等性
     */
    public List<String> processBatchIdempotent(List<Message> messages) {
        List<String> processedIds = new ArrayList<>();
        
        for (Message message : messages) {
            String messageId = message.getId();
            String businessType = message.getBusinessType();
            String businessKey = message.getBusinessKey();
            
            try {
                // 使用数据库唯一约束
                insertProcessRecord(messageId, businessType, businessKey);
                
                // 处理消息
                processMessage(message);
                processedIds.add(messageId);
                
            } catch (DuplicateKeyException e) {
                // 已处理，跳过
                System.out.println("跳过已处理消息: " + messageId);
            }
        }
        
        return processedIds;
    }
    
    private void insertProcessRecord(String messageId, String businessType, 
                                    String businessKey) {
        String sql = "INSERT INTO message_processed (message_id, business_type, " +
                    "business_key, processed_time) VALUES (?, ?, ?, NOW())";
        
        jdbcTemplate.update(sql, messageId, businessType, businessKey);
    }
    
    private boolean isMessageProcessed(String messageId, String businessType, 
                                      String businessKey) {
        String sql = "SELECT COUNT(*) FROM message_processed WHERE " +
                    "(message_id = ?) OR " +
                    "(business_type = ? AND business_key = ?)";
        
        Integer count = jdbcTemplate.queryForObject(sql, Integer.class, 
            messageId, businessType, businessKey);
        
        return count != null && count > 0;
    }
    
    private void rollbackProcessRecord(String messageId) {
        // 删除处理记录
        String sql = "DELETE FROM message_processed WHERE message_id = ?";
        jdbcTemplate.update(sql, messageId);
    }
    
    private void processMessage(Message message) {
        // 处理消息逻辑
    }
    
    /**
     * 消息实体
     */
    @Data
    public static class Message {
        private String id;
        private String businessType;
        private String businessKey;
        private String content;
    }
}
```

## 三、 分布式幂等性方案

### 1. Redis分布式锁方案

```
@Component
public class RedisDistributedIdempotent {
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    /**
     * 基于Redis SETNX的幂等性
     */
    public boolean processWithRedisLock(String messageId, String businessKey, 
                                       Supplier<Boolean> processor) {
        
        String lockKey = "idempotent:lock:" + businessKey;
        String processedKey = "idempotent:processed:" + messageId;
        
        // 1. 检查是否已处理
        Boolean processed = redisTemplate.hasKey(processedKey);
        if (Boolean.TRUE.equals(processed)) {
            System.out.println("消息已处理（Redis）: " + messageId);
            return false;
        }
        
        // 2. 获取分布式锁
        Boolean lockAcquired = redisTemplate.opsForValue()
            .setIfAbsent(lockKey, messageId, 30, TimeUnit.SECONDS);
        
        if (Boolean.FALSE.equals(lockAcquired)) {
            // 获取锁失败，可能其他实例正在处理
            System.out.println("获取锁失败: " + businessKey);
            return false;
        }
        
        try {
            // 3. 双重检查
            processed = redisTemplate.hasKey(processedKey);
            if (Boolean.TRUE.equals(processed)) {
                System.out.println("双重检查: 消息已处理");
                return false;
            }
            
            // 4. 执行业务逻辑
            boolean result = processor.get();
            
            if (result) {
                // 5. 标记为已处理
                redisTemplate.opsForValue()
                    .set(processedKey, "1", 1, TimeUnit.HOURS);
            }
            
            return result;
            
        } finally {
            // 6. 释放锁
            redisTemplate.delete(lockKey);
        }
    }
    
    /**
     * 使用Redisson分布式锁
     */
    public boolean processWithRedissonLock(String messageId, String businessKey, 
                                          Supplier<Boolean> processor, 
                                          RedissonClient redissonClient) {
        
        String lockKey = "idempotent:lock:" + businessKey;
        String processedKey = "idempotent:processed:" + messageId;
        
        // 检查是否已处理
        if (redissonClient.getBucket(processedKey).isExists()) {
            return false;
        }
        
        RLock lock = redissonClient.getLock(lockKey);
        
        try {
            // 尝试获取锁，最多等待5秒，锁持有30秒
            boolean locked = lock.tryLock(5, 30, TimeUnit.SECONDS);
            
            if (!locked) {
                System.out.println("获取Redisson锁失败");
                return false;
            }
            
            // 双重检查
            if (redissonClient.getBucket(processedKey).isExists()) {
                return false;
            }
            
            // 执行业务
            boolean result = processor.get();
            
            if (result) {
                redissonClient.getBucket(processedKey)
                    .set("1", 1, TimeUnit.HOURS);
            }
            
            return result;
            
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            return false;
        } finally {
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }
    
    /**
     * Lua脚本原子操作
     */
    public boolean processWithLuaScript(String messageId, String businessKey, 
                                       Supplier<Boolean> processor) {
        
        String luaScript = 
            "local processedKey = KEYS[1] " +
            "local lockKey = KEYS[2] " +
            "local messageId = ARGV[1] " +
            "local expireTime = ARGV[2] " +
            " " +
            "-- 检查是否已处理 " +
            "if redis.call('exists', processedKey) == 1 then " +
            "    return 0 " +
            "end " +
            " " +
            "-- 尝试获取锁 " +
            "local lockAcquired = redis.call('setnx', lockKey, messageId) " +
            "if lockAcquired == 0 then " +
            "    return -1 " +
            "end " +
            " " +
            "-- 设置锁过期时间 " +
            "redis.call('expire', lockKey, 30) " +
            " " +
            "-- 执行业务逻辑（这里需要外部执行） " +
            "-- 标记为已处理 " +
            "redis.call('setex', processedKey, expireTime, '1') " +
            " " +
            "-- 释放锁 " +
            "redis.call('del', lockKey) " +
            " " +
            "return 1";
        
        String processedKey = "idempotent:processed:" + messageId;
        String lockKey = "idempotent:lock:" + businessKey;
        
        // 执行Lua脚本
        DefaultRedisScript<Long> script = new DefaultRedisScript<>();
        script.setScriptText(luaScript);
        script.setResultType(Long.class);
        
        List<String> keys = Arrays.asList(processedKey, lockKey);
        Long result = redisTemplate.execute(script, keys, messageId, "3600");
        
        if (result == null || result == 0) {
            System.out.println("消息已处理（Lua）");
            return false;
        } else if (result == -1) {
            System.out.println("获取锁失败（Lua）");
            return false;
        }
        
        // 执行业务逻辑
        return processor.get();
    }
    
    /**
     * 批量幂等性处理
     */
    public Map<String, Boolean> processBatchWithRedis(List<Message> messages) {
        Map<String, Boolean> results = new HashMap<>();
        
        // 使用pipeline批量检查
        List<Object> pipelineResults = redisTemplate.executePipelined(
            new RedisCallback<Object>() {
                @Override
                public Object doInRedis(RedisConnection connection) throws DataAccessException {
                    for (Message message : messages) {
                        String key = "idempotent:processed:" + message.getId();
                        connection.exists(key.getBytes());
                    }
                    return null;
                }
            }
        );
        
        // 处理未处理的消息
        for (int i = 0; i < messages.size(); i++) {
            Message message = messages.get(i);
            Boolean processed = (Boolean) pipelineResults.get(i);
            
            if (Boolean.TRUE.equals(processed)) {
                results.put(message.getId(), false);
            } else {
                // 处理消息
                boolean success = processMessage(message);
                results.put(message.getId(), success);
                
                if (success) {
                    // 标记为已处理
                    String key = "idempotent:processed:" + message.getId();
                    redisTemplate.opsForValue().set(key, "1", 1, TimeUnit.HOURS);
                }
            }
        }
        
        return results;
    }
}
```

## 四、 业务层面的幂等性

### 1. 订单系统幂等性

```
@Component
public class OrderSystemIdempotent {
    
    @Autowired
    private OrderRepository orderRepository;
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    /**
     * 创建订单幂等性
     */
    public Order createOrderIdempotent(OrderRequest request) {
        String orderNo = request.getOrderNo();
        
        // 1. 检查Redis幂等性令牌
        String idempotentKey = "order:create:" + orderNo;
        Boolean notExists = redisTemplate.opsForValue()
            .setIfAbsent(idempotentKey, "processing", 5, TimeUnit.MINUTES);
        
        if (Boolean.FALSE.equals(notExists)) {
            // 正在处理或已处理
            throw new BusinessException("订单正在处理中，请勿重复提交");
        }
        
        try {
            // 2. 检查数据库是否已存在
            Optional<Order> existingOrder = orderRepository.findByOrderNo(orderNo);
            if (existingOrder.isPresent()) {
                return existingOrder.get();
            }
            
            // 3. 创建订单
            Order order = createOrder(request);
            
            // 4. 更新幂等性令牌为完成
            redisTemplate.opsForValue().set(idempotentKey, "completed", 1, TimeUnit.HOURS);
            
            return order;
            
        } catch (Exception e) {
            // 5. 处理失败，删除幂等性令牌
            redisTemplate.delete(idempotentKey);
            throw e;
        }
    }
    
    /**
     * 支付回调幂等性
     */
    public boolean processPaymentCallback(String paymentNo, String orderNo, 
                                         BigDecimal amount, String status) {
        
        // 幂等性Key
        String idempotentKey = "payment:callback:" + paymentNo;
        
        // 使用Lua脚本保证原子性
        String luaScript = 
            "local key = KEYS[1] " +
            "local status = ARGV[1] " +
            " " +
            "-- 检查是否已处理 " +
            "local processed = redis.call('get', key) " +
            "if processed then " +
            "    return 0 " +
            "end " +
            " " +
            "-- 标记为处理中 " +
            "redis.call('setex', key, 300, 'processing') " +
            "return 1";
        
        DefaultRedisScript<Long> script = new DefaultRedisScript<>();
        script.setScriptText(luaScript);
        script.setResultType(Long.class);
        
        Long result = redisTemplate.execute(script, List.of(idempotentKey), status);
        
        if (result == null || result == 0) {
            // 已处理
            System.out.println("支付回调已处理: " + paymentNo);
            return false;
        }
        
        try {
            // 处理支付回调
            return processPayment(orderNo, paymentNo, amount, status);
            
        } finally {
            // 更新处理状态
            if (status.equals("SUCCESS")) {
```

```
     */
    public <T> T monitorIdempotentCheck(Supplier<T> checker, String businessType) {
        long startTime = System.currentTimeMillis();
        
        try {
            T result = checker.get();
            
            long duration = System.currentTimeMillis() - startTime;
            idempotentCheckCounter.increment();
            idempotentCheckTimer.record(duration, TimeUnit.MILLISECONDS);
            
            // 记录指标
            IdempotentMetrics metrics = metricsMap.computeIfAbsent(

            return result;
            
        } catch (Exception e) {

        }
    }

            return metrics.getStats();

     * 健康检查
     */
    public Health idempotentHealthCheck() {
        Map<String, Object> details = new HashMap<>();
        
        // 计算重复率
        long totalChecks = idempotentCheckCounter.count();
        long totalDuplicates = duplicateMessageCounter.count();
        double duplicateRate = totalChecks > 0 ? 
            (double) totalDuplicates / totalChecks : 0.0;
        
        details.put("totalChecks", totalChecks);
        details.put("totalDuplicates", totalDuplicates);
        details.put("duplicateRate", duplicateRate);
        
        metricsMap.forEach((type, metrics) -> {
            IdempotentStats stats = metrics.getStats();
            details.put(type + ".duplicateRate", stats.getDuplicateRate());
            details.put(type + ".avgCheckTime", stats.getAvgCheckTime());
        });
        
        if (duplicateRate > 0.1) { // 重复率超过10%
            return Health.down()
                .withDetail("error", "重复率过高: " + duplicateRate)

                .build();
        }
```

## 八、 常见问题

### Q1: 幂等性有效期设置多久？

- 临时操作：几分钟到几小时
    
- 普通业务：1-7天
    
- 重要业务：30天
    
- 金融业务：永久或按监管要求
    

### Q2: 幂等性存储选型？

- Redis：高性能，适合临时幂等
    
- 数据库：可靠，适合重要业务
    
- 内存：快速，但重启丢失
    
- 混合：多级存储，兼顾性能可靠
    

### Q3: 如何处理幂等性数据清理？

- 按时间自动清理
    
- 按数量限制清理
    
- 业务完成后清理
    
- 定期归档清理
    

### Q4: 分布式环境下如何保证幂等性？

- 分布式锁
    
- 数据库唯一约束
    
- Redis原子操作
    
- 消息队列顺序消费
    

业务幂等性是分布式系统的基石，合理设计可以避免重复处理、数据不一致等问题，提高系统稳定性和可靠性。
