`IService`是 **MyBatis Plus 框架提供的、位于 Service 层的通用接口**。它是 `BaseMapper`在 Service 层的“镜像”和扩展，旨在为**业务逻辑层**提供一套开箱即用、功能强大的通用 CRUD 及批处理操作方法，是 MyBatis Plus 实现**代码复用**和**规范分层**的核心组件之一。

简单来说，它是框架为你准备好的 Service 层接口模板，你的 Service 接口只需继承它，你的 Service 实现类只需继承其实现类 `ServiceImpl`，就立刻拥有了大量现成的方法，无需再编写基础 CRUD 代码。

### 一、核心概念与关系

```
// 1. 你的 Mapper 层接口
public interface UserMapper extends BaseMapper<User> {}

// 2. 你的 Service 层接口 (继承 IService)
public interface IUserService extends IService<User> {
    // 这里可以定义自己独特的业务方法
    UserVO getUserWithDetail(Long id);
}

// 3. 你的 Service 层实现类 (继承 ServiceImpl，并实现你自己的接口)
@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, User> 
                             implements IUserService {
    // 自动拥有了 IService 的所有方法
    // 只需实现自定义的业务方法
    @Override
    public UserVO getUserWithDetail(Long id) {
        // ... 自定义业务逻辑
    }
}
```

**核心关系图**：

`IYourService`(你的接口) 继承 → `IService<T>`(MP 提供的通用接口)

`YourServiceImpl`(你的实现类) 继承 → `ServiceImpl<M, T>`(MP 提供的通用实现类)，并实现 `IYourService`

### 二、主要功能与优势

`IService`的方法可以看作是 `BaseMapper`方法的“增强版”和“批处理版”，主要解决了以下问题：

**1. 批量操作（核心优势）**

这是 `BaseMapper`所不具备的，`IService`提供了高效的批量插入、更新、删除。

```
// 批量插入 (效率远高于循环调用 save)
boolean saveBatch(Collection<T> entityList);
boolean saveBatch(Collection<T> entityList, int batchSize); // 指定批次大小

// 批量更新
boolean updateBatchById(Collection<T> entityList);

// 批量保存或更新 (根据ID判断是新增还是更新)
boolean saveOrUpdateBatch(Collection<T> entityList);

// 根据ID批量删除
boolean removeByIds(Collection<?> idList);
```

**2. 链式查询（Lambda 风格）**

提供更直观的链式调用方式，用于构建查询条件。

```
// 链式查询
User one = userService.lambdaQuery()
        .eq(User::getName, "张三")
        .gt(User::getAge, 18)
        .one();

List<User> list = userService.lambdaQuery()
        .like(User::getName, "王")
        .list();

// 链式更新
boolean updated = userService.lambdaUpdate()
        .eq(User::getStatus, 0)
        .set(User::getStatus, 1)
        .update();
```

**3. 更灵活的查询与计数**

```
// 返回 Map
Map<String, Object> getMap(Wrapper<T> queryWrapper);

// 返回对象列表（任意条件）
List<T> list(Wrapper<T> queryWrapper);

// 返回对象列表（按ID集合）
Collection<T> listByIds(Collection<?> idList);

// 计数
long count(Wrapper<T> queryWrapper);
```

**4. 获取基础Mapper**

可以在 Service 层直接拿到对应的 Mapper 实例，进行更底层的操作。

```
UserMapper baseMapper = getBaseMapper();
```

### 三、使用示例

**1. 基本 CRUD 操作**

```
@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, User> 
                             implements IUserService {
    // 无需编写任何代码，已拥有下列方法：
    // save, saveBatch, removeById, removeByIds, updateById, updateBatchById, 
    // getById, listByIds, list, page, count 等
}

// 在Controller或其它Service中使用
@RestController
@RequestMapping("/user")
public class UserController {
    @Autowired
    private IUserService userService;
    
    public void someMethod() {
        // 1. 增
        User user = new User();
        user.setName("李四");
        userService.save(user);
        
        // 2. 批量增
        List<User> userList = new ArrayList<>();
        // ... 添加多个user
        userService.saveBatch(userList, 1000); // 每1000条提交一次
        
        // 3. 改
        user.setAge(30);
        userService.updateById(user);
        
        // 4. 批量改
        userService.updateBatchById(userList);
        
        // 5. 删
        userService.removeById(1L);
        userService.removeByIds(Arrays.asList(1L, 2L, 3L));
        
        // 6. 查
        User u = userService.getById(1L);
        List<User> all = userService.list();
        long count = userService.count();
    }
}
```

**2. 链式操作示例**

```
// 查询：年龄大于20，状态为1的用户，按创建时间倒序，取前10条
List<User> users = userService.lambdaQuery()
        .gt(User::getAge, 20)
        .eq(User::getStatus, 1)
        .orderByDesc(User::getCreateTime)
        .last("LIMIT 10") // 可以拼接原生SQL片段
        .list();

// 更新：将所有名字包含“test”的用户的邮箱清空
boolean success = userService.lambdaUpdate()
        .like(User::getName, "test")
        .set(User::getEmail, null)
        .update();

// 查询一个：名字为“张三”的用户
User zhangsan = userService.lambdaQuery()
        .eq(User::getName, "张三")
        .one(); // 查不到或多条会抛异常
User maybeNull = userService.lambdaQuery()
        .eq(User::getName, "张三")
        .oneOpt() // 返回 Optional，更安全
        .orElse(null);
```

**3. 分页查询**

```
// 构建分页参数（第2页，每页10条）
Page<User> pageParam = new Page<>(2, 10);
// 构建查询条件
LambdaQueryWrapper<User> wrapper = Wrappers.lambdaQuery();
wrapper.gt(User::getAge, 18);

// 执行分页查询
IPage<User> pageResult = userService.page(pageParam, wrapper);

// 获取结果
List<User> records = pageResult.getRecords(); // 当前页数据
long total = pageResult.getTotal();           // 总记录数
long pages = pageResult.getPages();           // 总页数
```

### 四、与 BaseMapper 的关系与选择

|特性|`BaseMapper`(Mapper 层)|`IService`(Service 层)|
|---|---|---|
|**定位**​|数据访问层，直接操作数据库|业务逻辑层，封装业务操作|
|**核心方法**​|单条 CRUD、条件查询|**批量操作**、链式查询、业务聚合|
|**事务**​|通常不直接处理事务|**天然是事务的边界**，可在方法上加 `@Transactional`|
|**使用场景**​|在 Service 内部调用，或简单场景|**Controller 或其他 Service 应调用的对象**​|
|**代码生成**​|代码生成器可生成|代码生成器可生成其实现类|

**最佳实践**：

- **永远**在 Service 层进行事务管理（`@Transactional`）。
    
- Controller 应调用 `IUserService`，而不是 `UserMapper`。
    
- 简单的单条操作，用 `userService.getById(id)`和 `userMapper.selectById(id)`效果一样，但前者保持了调用层的一致性。
    
- **只要涉及批量操作，必须使用 `IService`的方法**（如 `saveBatch`），其内部做了性能优化（例如合并 SQL 语句）。
    

### 五、重要的实现类：`ServiceImpl<M, T>`

这是 `IService`的默认实现，你的 Service 实现类需要继承它。

- **泛型 `<M, T>`**：
    
    - `M`：你的 Mapper 接口类型（如 `UserMapper`）。
        
    - `T`：你的实体类类型（如 `User`）。
        
    
- **它内部**通过 `getBaseMapper()`方法持有了你的 Mapper，所有对数据库的操作最终都委托给了 `BaseMapper`。
    
- 继承后，你可以通过 `this.getBaseMapper()`获取到原生的 Mapper 对象，以调用其自定义方法。
    

### 总结

**`IService`是什么？**

它是一个**服务层的通用接口模板**，是 MyBatis Plus 为规范项目架构、提升开发效率而设计的关键组件。它通过：

1. **提供大量现成的批处理和增强方法**，避免了 Service 层的重复编码。
    
2. **强制约束了项目分层**（Controller -> Service -> Mapper），使架构更清晰。
    
3. **集成链式操作**，让条件构建在 Service 层也能流畅书写。
    

**何时使用？**

在绝大多数 MyBatis Plus 项目中，你的 Service 接口都应继承 `IService`，实现类继承 `ServiceImpl`。这是使用 MyBatis Plus 开发业务功能的**标准做法和最佳起点**。对于极其简单的场景，你可以选择直接注入 Mapper 使用，但会失去批处理、链式操作等便利，且不利于项目长期维护和规范。
